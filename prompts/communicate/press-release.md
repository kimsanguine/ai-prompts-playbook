# 📰 비전 프레스 릴리즈 (Visionary Press Release)

> 예상 소요: 5분 (Quick) / 20분 (Deep) / 30분+ (Build)
> 워크플로우: ← `stakeholder-alignment.md` → `storytelling.md`

---

## 🎯 이 프롬프트가 해결하는 문제

AI 기능을 출시했을 때 "어떻게 고객에게 설명할 건가?"라는 질문이 나온다. 일반적인 프레스 릴리즈는 "제품 출시 소식"만 전달한다. **Amazon Working Backwards 방식의 프레스 릴리즈는 "왜 이 제품을 만들었는가? 고객의 어떤 문제를 해결하는가? 미래는 어떻게 될 건가?"를 먼저 작성한다.** 이는 고객에게 비전을 전달하고, 팀 내에서도 목표를 명확히 한다.

---

## 📥 이전 프롬프트에서 받는 입력

> `positioning-strategy.md`에서 수립한 포지셔닝을 Working Backwards 프레스 릴리즈로 구체화할 때 연결됩니다.

이전 프롬프트 결과에서 **포지셔닝 문장, 타겟 고객, 핵심 차별점**을 가져옵니다.

## ⚡ Layer 1: Quick Start

> **이 레이어가 하는 일: 전문가의 눈 빌리기**
> 전문가가 이 분야를 볼 때 쓰는 프레이밍을 빌려줍니다. AI를 처음 쓰는 분, 비개발자, 학생 — 5분이면 제품 출시 보도자료의 핵심 구조를 작성할 수 있습니다.

### 📌 사용법

1. **ChatGPT, Claude, Gemini 중 아무 AI 챗봇**을 열고 새 대화를 시작하세요
2. 아래 프롬프트를 **통째로 복사**해서 AI에 붙여넣으세요
3. `{중괄호}` 안의 내용만 여러분의 상황에 맞게 바꾸세요
4. Enter를 누르면 끝!

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


```
당신은 PM 커뮤니케이션 전문가다.

Amazon Working Backwards 방식으로 프레스 릴리즈를 작성하세요.

구조:
1. 제목: "핵심 약속"을 한 문장으로
2. 부제: "누가, 어떤 문제"를 해결하는가?
3. Opening: "고객의 목소리"로 시작 (고객 인용문)
4. 문제: 고객이 현재 겪는 문제 (구체적 수치)
5. 해결: 우리의 솔루션과 차별점
6. 미래: "이제 고객들은..." (기대되는 변화)
7. Quotes: 경영진, 고객, 전문가 의견

각 섹션별로 명확하고 감정을 담아 작성.
```

💡 **Quick Tips:**
- "일반적인 프레스 릴리즈"보다 "Working Backwards" 방식이 훨씬 강력
- 고객의 실제 인용문 (가능하면 베타 사용자의 진짜 피드백)
- 미래 시제로: "이제...", "앞으로..." → 비전을 제시


### ⚠️ 이런 경우는요?

| 상황 | 대처법 |
|------|--------|
| 내부용 (Working Backwards) | 외부 발표가 아닌 내부 방향 정렬 도구로 사용. 기밀 정보 포함 가능 |
| 부정적 뉴스 (보안 사고 등) | 위기 커뮤니케이션 템플릿을 사용하세요. 이 프롬프트는 긍정적 발표용 |
| 글로벌 발표 | 문화별 프레이밍 차이를 고려하세요 (미국: 혁신, 한국: 신뢰/실적) |
| B2B 보도자료 | 고객 사례와 ROI 수치를 포함하면 설득력이 높아집니다 |

## 🧠 Layer 2: Deep Dive

> **이 레이어는 누구를 위한 건가요?**
> PM, 기획자, 마케터 — Amazon의 Working Backwards 방식으로 보도자료를 작성하고 싶은 분입니다.
"제품이 완성되면 어떤 기사가 나올까?"부터 역설계하고 싶다면 이걸 써보세요.

### 📌 진행 방식

이 프롬프트는 **5단계 CoT(Chain of Thought)** 구조입니다. 두 가지 방식으로 쓸 수 있습니다:

**방식 A: 한 번에 넣기 (추천)**
- 아래 전체를 통째로 복사해서 AI에 붙여넣으세요
- AI가 모든 단계를 순서대로 실행합니다
- 결과가 한 번에 나옵니다 (길지만 완결적)

**방식 B: 단계별로 넣기 (더 깊은 분석)**
- 1-2단계를 먼저 넣고 결과를 확인하세요
- 결과가 만족스러우면 다음 단계를 이어서 넣으세요
- 중간에 수정이 필요하면 해당 단계를 수정 요청할 수 있습니다
- 시간이 더 걸리지만, 각 단계를 직접 검증할 수 있습니다

**`<!-- 💬 -->` 교육 주석이란?**
- 프롬프트 안에 `<!-- 💬 ... -->` 형태의 주석이 있습니다
- 이건 **여러분을 위한 설명**이지, AI에게 보내는 지시가 아닙니다
- AI에 붙여넣을 때 포함해도 되고 빼도 됩니다 (결과에 영향 없음)
- "왜 이렇게 설계했는가?"를 이해하면 더 좋은 프롬프트를 만들 수 있습니다

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


### 역할 설정

당신은 **AI 제품의 비전 스토리텔러**이다. 당신의 역할:

1. **Working Backwards 마인드셋**: 고객이 먼저, 제품은 나중
2. **감정 전달**: "기술 스펙"이 아니라 "고객의 변화"를 이야기
3. **신뢰성**: 구체적 데이터와 실제 고객 목소리

### 목표

최종 아웃풋:
- **Working Backwards 프레스 릴리즈** (고객 관점)
- **미디어 키트** (스크린샷, 데이터, 인용문)
- **FAQ** (기자 질문 대비)
- **한국 시장 특화 버전** (문화적 맥락)

### Reasoning Strategy

**Step 1: 고객 문제 정의 (Working Backwards 시작)**

```
❌ 제품 중심:
"우리는 AI 요약 기능을 만들었다"

✅ 고객 중심:
"고객들의 문제: 매일 200페이지 문서를 읽느라 의사결정이 지연됨"
"이제: AI가 핵심 5줄로 정리해주니, 의사결정 시간이 80% 단축됨"
```

**Step 2: 고객 인용문 확보**

프레스 릴리즈의 가장 강력한 부분은 실제 고객의 목소리:

```
좋은 인용문:
"사람이 읽으면 30분 걸리는 계약서를
AI 요약으로 2분 만에 이해할 수 있으니,
비즈니스 속도가 정말 달라집니다." - 고객 CTO

나쁜 인용문:
"우리는 이 기능이 매우 만족스럽습니다." - 경영진
→ 제너릭, 감정 없음
```

<!-- 💬 [논쟁 지점] 실제 고객이 없을 때는?

     방법2: 가상 인용 작성 → 실제 피드백으로 검증 (차선)
     방법3: 기자간담회나 보도 후 실제 고객으로 update (위험)
     한국 미디어: 거짓 인용문에 민감 (아웃스탠딩, 요즘IT 등) -->

**Step 3: 미래 상태 (Future-looking statements)**

```
지금: "문서 읽기에 시간 낭비"
→ 미래: "AI가 맥락을 이해하고, 의사결정을 제안"

이렇게 표현:
"앞으로 고객들은 더 이상 단순 정보 수집에 시간을 쓰지 않고,
전략적 의사결정과 혁신에 집중할 수 있게 될 것입니다."
```

**Step 4: 한국 시장 특화 (문화적 맥락)**

```
한국 기업 문화 반영:
- 빠른 의사결정 → "의사결정 속도 중시"
- 다단계 검증 → "담당자별 검토 자동화"
- 신뢰 중시 → "AI 신뢰도 표시로 검증 용이"
- 보고선 최적화 → "결재라인 단축"
- 린품의 문화 → "회의 시간 단축"

예시:
"한국 기업들은 신속한 의사결정을 위해 여러 단계의 검토를 거칩니다.
이제 AI가 초안을 검토하고 신뢰도까지 표시해주니,
담당자들의 검증 프로세스가 훨씬 수월해집니다."

한국 PR 전략:
- 대상: 아웃스탠딩, 요즘IT, eo (IT 전문 매체)
- 각도: "한국 스타트업의 속도" + "대기업의 신뢰성" 조화
- 기자간담회: "한국 기업의 실제 고통" 중심으로 진행
```

### Working Backwards 프레스 릴리즈 템플릿

```markdown
# [회사명], AI 기반 문서 분석으로 업무 시간 80% 단축

## 개요 (Subheading)
서비스 문서, 계약서, 보고서를 2초 만에 핵심 5줄로 정리해주는 AI 요약 기능 출시

---

## 고객의 문제 (Opening with Customer Problem)

"저희 팀은 매일 100페이지 이상의 법무 계약서를 검토합니다.
각 계약서마다 30분씩 읽고 정리하다 보니,
정작 중요한 의사결정에는 시간이 부족합니다."

- 현 상황: 한국 기업 임직원이 매일 평균 2시간을 문서 읽기에 소비 (IDC 조사)
- 영향: 의사결정 지연, 리드 타임 증가, 직원 피로

---

## 우리의 해결책

오늘 [회사명]은 AI 요약 기능을 출시합니다.
이는 고객이 제출한 계약서, 보고서, 이메일을 2초 만에 핵심으로 정리하고,
AI 신뢰도까지 표시해주어, 사용자가 자신감 있게 의사결정할 수 있게 합니다.

### 차별점
1. **초고속**: 200페이지 문서도 2초
2. **신뢰도**: AI 신뢰도 점수 (7.2/10 같이) 표시
3. **한국 특화**: 한국 계약서, 정부 공문 최적화

---

## 기대되는 변화

### 이제 고객들은...
- 문서 검토 시간을 80% 단축할 수 있습니다
- "이 계약서의 핵심이 뭐지?"에 2초 안에 답할 수 있습니다
- AI의 결과를 신뢰도 점수로 판단할 수 있으니, 검증 부담이 줄어듭니다

### 조직 차원
- 의사결정 속도 ↑ (리드 타임 -80%)
- 직원 생산성 ↑ (고부가가치 일에 집중)
- 규정 준수 ↑ (자동 감시 기능)
```

### FAQ (기자 대비용)

```markdown
# AI 요약 기능 FAQ

## Q1. 정말 95% 정확도인가? 어떻게 검증했나?
A: 한국 법무 서류 500개를 변호사가 직접 검증했고,
95%의 경우 AI 요약이 "정확하고 누락이 없다"고 평가했습니다.
상세 벤치마크는 [링크]를 참고하세요.

## Q2. 데이터 보안은? 내 계약서가 OpenAI에 저장되나?
A: 절대 아닙니다. 우리는 자체 인프라에서만 처리하며,
고객 데이터는 24시간 후 자동 삭제됩니다.
GDPR, CCPA 준수합니다.

## Q3. 신뢰도 점수 7.2/10이 뭐야? 왜 완벽하지 않은가?
A: AI는 완벽할 수 없고, 투명성이 중요합니다.
신뢰도 점수가 낮으면 사람이 직접 검토해야 한다는 신호입니다.
이는 "AI를 맹신하지 말라"는 우리의 철학입니다.

## Q4. 한국어만 지원하나?
A: 현재는 한국어, 영어를 지원합니다.
Q3에 중국어, 일본어가 추가될 예정입니다.

## Q5. 비용이 얼마나?
A: Free (월 10개), Starter ($99/월, 무제한), Pro (Custom pricing)
상세 가격 책정은 [가격 페이지]를 참고하세요.

## Q6. 경쟁사(Notion, ChatGPT)와 뭐가 다른가?
A:
- Notion: 일반적인 요약만 가능 (한국어 최적화 X)
- ChatGPT: 신뢰도 표시 없음, 비용 불명확
- 우리: 한국 계약서 특화 + 신뢰도 표시 + 명확한 가격

## Q7. 언제 GA 출시인가?
A: Beta 2026-03-15, GA 2026-06-15 예정입니다.
Beta 동안 고객 피드백을 반영해 기능을 개선합니다.

## Q8. 사람 일자리를 빼앗지 않나?
A: 아닙니다. 이 도구는 사람의 능력을 강화합니다.
법무팀이 단순 읽기가 아니라 전략적 검토에 집중하게 됩니다.
실제 고객들은 팀 규모를 늘리지 않고 업무량만 증가시켰습니다.
```

💡 **Deep Tips:**
- **고객 인용문**: 진짜 고객의 진짜 목소리 (기획자의 입장이 아니라 사용자의 임팩트)
- **미래 지향**: "지금"이 아니라 "앞으로 고객들은" 식으로 비전 제시
- **한국 특화**: 글로벌 템플릿이 아니라 한국 기업 문화, 업무 방식 반영

## 🔧 Layer 3: Build & Automate

> **이 레이어는 누구를 위한 건가요?**
> AI 엔지니어, 개발자 — 제품 정보에서 자동으로 보도자료 초안을 생성합니다.

### 📌 시작 전에

**이 코드가 하는 일:**
Layer 1-2에서 수동으로 했던 과정을 Python 스크립트로 자동화합니다.

**왜 자동화하나?**
보도자료는 제품 출시, 업데이트, 파트너십 등에서 반복됩니다. 자동화하면 일관된 톤과 구조를 유지할 수 있습니다.

**전제조건:**
- Python 3.8+ 설치
- 필요한 패키지 설치 (코드 상단의 import 참고)
- Anthropic API 키 발급 → 환경변수 설정: `export ANTHROPIC_API_KEY=your_key`

> 📖 더 자세한 설정 방법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


### Working Backwards 프레스 릴리즈 생성기 (Python)

```python
# press_release_generator.py
from typing import List, Dict
from datetime import datetime

@dataclass
class PressReleaseComponent:
    """프레스 릴리즈 구성 요소"""
    customer_problem: str  # 고객의 현재 문제
    quantified_impact: str  # 구체적 수치 (시간, 비용 등)
    solution: str  # 우리의 솔루션
    differentiation: List[str]  # 차별점
    customer_quotes: List[Dict]  # 고객 인용문
    expected_outcomes: List[str]  # 기대되는 변화
    timeline: Dict  # 출시 일정

class WorkingBackwardsGenerator:
    """Working Backwards 프레스 릴리즈 생성"""

    def __init__(self, product_name: str, company_name: str):
        self.product_name = product_name
        self.company_name = company_name

    def generate_release(self, component: PressReleaseComponent) -> str:
        """Working Backwards 방식의 프레스 릴리즈 생성"""

        release = f"""
# {self.company_name}, AI 기능으로 고객 업무 {component.quantified_impact} 단축

## 개요
[제품 설명 한 줄]

---

## 고객의 문제

{component.customer_problem}

**현 상황의 영향:**
{component.quantified_impact}

---

## 우리의 해결책

오늘 {self.company_name}은 {self.product_name}을 출시합니다.

### 주요 특징
"""

        for i, diff in enumerate(component.differentiation, 1):
            release += f"{i}. {diff}\n"

        release += "\n---\n\n## 기대되는 변화\n\n### 이제 고객들은...\n"
        for outcome in component.expected_outcomes:
            release += f"- {outcome}\n"

        release += "\n---\n\n## 고객 목소리\n\n"
        for quote in component.customer_quotes:
            release += f"""
> "{quote['text']}"
> — {quote['name']}, {quote['company']}

"""

        release += f"""
---

## 출시 일정
"""
        for phase, date in component.timeline.items():
            release += f"- **{phase}**: {date}\n"

        return release

    def generate_executive_quote(self, ceo_name: str, company_vision: str) -> str:
        """경영진 인용문 생성"""
        return f"""
> "{company_vision}"
> — {ceo_name}, CEO of {self.company_name}
"""

    def generate_faq(self, common_questions: List[Dict]) -> str:
        """FAQ 생성"""
        faq = "# FAQ\n\n"
        for q_a in common_questions:
            faq += f"## Q: {q_a['question']}\nA: {q_a['answer']}\n\n"
        return faq


# 사용 예시
if __name__ == "__main__":
    component = PressReleaseComponent(
        customer_problem=(
            "한국 기업 임직원들은 매일 수십 개의 문서를 검토합니다. "
            "계약서, 보고서, 이메일을 읽고 정리하는 데 하루 2시간을 소비합니다."
        ),
        quantified_impact="80% 단축 (2시간 → 24분)",
        solution="Claude API 기반 AI 요약 기능",
        differentiation=[
            "초고속: 200페이지 문서도 2초",
            "신뢰도: AI 신뢰도 점수 표시",
            "한국 특화: 한국 계약서, 정부 공문 최적화",
        ],
        customer_quotes=[
            {
                "text": "계약서 검토 시간이 80% 단축되니, 팀이 전략 업무에 집중할 수 있습니다.",
                "name": "이순신",
                "company": "삼성 금융 센터",
            },
            {
                "text": "신뢰도 점수가 표시되니까, AI 결과를 맹신하지 않고 신뢰도 낮은 부분만 검토합니다.",
                "name": "유관순",
                "company": "한국 법무법인",
            },
        ],
        expected_outcomes=[
            "의사결정 속도 80% 향상",
            "직원 생산성 증가 (고부가가치 업무에 집중)",
            "규정 준수 자동화",
        ],
        timeline={
            "Beta": "2026-03-15",
            "GA": "2026-06-15",
            "Phase 2 (이미지 지원)": "2026-Q3",
            "Phase 3 (자동 액션)": "2026-Q4",

    generator = WorkingBackwardsGenerator("AI 요약 기능", "우리 회사")
    release = generator.generate_release(component)
```

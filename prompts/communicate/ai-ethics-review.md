# ⚖️ AI 윤리 리뷰 (AI Ethics Review)

> 예상 소요: 5분 (Quick) / 25분 (Deep) / 40분+ (Build)
> 워크플로우: ← `stakeholder-alignment.md` → 출시 체크리스트

---

## 🎯 이 프롬프트가 해결하는 문제

AI 기능을 출시하기 전에 **"이 AI가 윤리적으로 문제 없는가?"**를 검토해야 한다. 편향성(bias), 데이터 프라이버시, 불명확한 책임(accountability), 접근성 등 AI 특화 윤리 이슈를 체크하지 않으면, 출시 후 사용자 신뢰와 법적 문제로 이어진다.

---

## 📥 이전 프롬프트에서 받는 입력

> `stakeholder-alignment.md`에서 이해관계자 합의 후, 출시 전 윤리 검토 단계에서 연결됩니다.

이전 프롬프트 결과에서 **AI 기능명, 사용 데이터 유형, 대상 사용자**를 가져옵니다.

---

## ⚡ Layer 1: Quick Start

> **이 레이어가 하는 일: 전문가의 눈 빌리기**
> 전문가가 이 분야를 볼 때 쓰는 프레이밍을 빌려줍니다. AI를 처음 쓰는 분, 비개발자, 학생 — 5분이면 AI 기능의 핵심 윤리 체크리스트를 검토할 수 있습니다.

### 📌 사용법

1. **ChatGPT, Claude, Gemini 중 아무 AI 챗봇**을 열고 새 대화를 시작하세요
2. 아래 프롬프트를 **통째로 복사**해서 AI에 붙여넣으세요
3. `{중괄호}` 안의 내용만 여러분의 상황에 맞게 바꾸세요
4. Enter를 누르면 끝!

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


```
당신은 AI 윤리 검토자다.

{AI 기능}을 다음 항목별로 체크하세요:

1. 편향성(Bias): 특정 그룹에 불리하진 않나?
   예) 남성 이름은 높은 신용도, 여성 이름은 낮은 신용도

2. 데이터 프라이버시: 사용자 데이터를 안전하게 처리하는가?
   예) 계약서 내용이 모델 학습에 사용되진 않나?

3. 투명성(Transparency): 사용자가 AI의 결정을 이해할 수 있는가?
   예) "신뢰도 7.2/10"을 표시하는가?

4. 책임(Accountability): AI가 오류를 낸 경우, 누가 책임지는가?

5. 접근성(Accessibility): 모든 사용자(장애인 포함)가 쓸 수 있는가?

6. 노동 윤리: 이 AI가 사람의 일자리를 없애진 않나?
   (기능 강화 vs 인원 감축)

각 항목별로:
- 현황 (문제 있나?)
- 위험도 (심각한가?)
- 해결책 (어떻게 개선?)

구체적이고 실행 가능하게.
```

💡 **Quick Tips:**
- 윤리 체크는 "도덕" 문제가 아니라 "비즈니스 위험" 문제
- 편향성 발견 시 → 학습 데이터 재검토 → 모델 재학습
- 투명성 부족 시 → 신뢰도 표시 추가 → 사용자 신뢰 증가


### ⚠️ 이런 경우는요?

| 상황 | 대처법 |
|------|--------|
| AI 윤리가 처음 | 체크리스트 형태의 Layer 1만 먼저 사용하세요 |
| 법적 요구사항이 불확실 | 법무팀과 협의하세요. AI 윤리 ≠ 법적 준수이지만 겹치는 부분이 많습니다 |
| 팀에 윤리 전문가가 없음 | 외부 윤리 리뷰 서비스를 고려하거나, 산업 가이드라인을 따르세요 |
| 시간이 부족 | 최소한 "편향 체크 + 프라이버시 체크"만이라도 하세요 |

## 🧠 Layer 2: Deep Dive

> **이 레이어는 누구를 위한 건가요?**
> PM, 기획자, 마케터 — AI 윤리를 체계적으로 검토하고 리스크를 사전에 파악하고 싶은 분입니다.
편향, 프라이버시, 설명가능성까지 포괄적으로 점검하고 싶다면 이걸 써보세요.

### 📌 진행 방식

이 프롬프트는 **6단계 CoT(Chain of Thought)** 구조입니다. 두 가지 방식으로 쓸 수 있습니다:

**방식 A: 한 번에 넣기 (추천)**
- 아래 전체를 통째로 복사해서 AI에 붙여넣으세요
- AI가 모든 단계를 순서대로 실행합니다
- 결과가 한 번에 나옵니다 (길지만 완결적)

**방식 B: 단계별로 넣기 (더 깊은 분석)**
- 1-2단계를 먼저 넣고 결과를 확인하세요
- 결과가 만족스러우면 다음 단계를 이어서 넣으세요
- 중간에 수정이 필요하면 해당 단계를 수정 요청할 수 있습니다
- 시간이 더 걸리지만, 각 단계를 직접 검증할 수 있습니다

**`<!-- 💬 -->` 교육 주석이란?**
- 프롬프트 안에 `<!-- 💬 ... -->` 형태의 주석이 있습니다
- 이건 **여러분을 위한 설명**이지, AI에게 보내는 지시가 아닙니다
- AI에 붙여넣을 때 포함해도 되고 빼도 됩니다 (결과에 영향 없음)
- "왜 이렇게 설계했는가?"를 이해하면 더 좋은 프롬프트를 만들 수 있습니다

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


### 역할 설정

당신은 **AI 제품의 윤리 감시자(Ethics Guardian)**이다. 당신의 역할:

1. **편향성 감지**: 모델이 특정 그룹을 차별하는가?
2. **프라이버시 보호**: 사용자 데이터가 안전한가?
3. **투명성 확보**: "AI가 왜 이 결정을 내렸는가?"를 설명할 수 있는가?
4. **책임 명확화**: 문제 발생 시 누가 어떻게 책임질 건가?

### 목표

최종 아웃풋:
- **AI 윤리 체크리스트** (6개 항목별 평가)
- **위험도 평가** (높음/중간/낮음)
- **개선 액션 플랜** (구체적 해결책)
- **윤리 정책 문서** (법무/컴플라이언스용)

<!-- 💬 [전문가의 눈] AI 윤리는 "도덕" 문제가 아니라 "비즈니스 위험" 문제
     편향성 AI → 특정 그룹 차별 → 소송 → 손실
     프라이버시 위반 → 규정 위반 → 과징금 → 신뢰 저하
     투명성 부족 → 사용자 불만 → 서비스 중단 압박

     결론: 윤리 체크리스트는 비용이 아니라 리스크 관리 도구 -->

### Reasoning Strategy

**Step 1: 편향성(Bias) 검토**

```
❓ 우리 AI는 특정 그룹을 차별하는가?

예시 체크:
- 성별 편향: 여성 이름의 신용도 점수가 평균적으로 낮은가?
- 나이 편향: 고령자 신청이 자동 거절되는가?
- 지역 편향: 특정 지역은 높은 위험도 평가?
- 학력 편향: 대졸만 높은 점수?

검증 방법:
1. 테스트 데이터 구성: 각 그룹별 동등한 샘플 (여성 100명, 남성 100명)
2. 성능 비교: 그룹별 정확도/신뢰도 비교
3. 편향성 메트릭: Demographic Parity, Equal Opportunity 계산
4. 발견 시 액션:
   - 원인: 학습 데이터 불균형? 모델 아키텍처?
   - 해결: 데이터 augmentation, Fair ML 알고리즘 적용
```

**Step 2: 데이터 프라이버시 검토**

```
❓ 우리는 사용자 데이터를 안전하게 처리하는가?

체크리스트:
□ 데이터 수집 동의: 사용자가 명시적으로 동의했는가?
□ 목적 제한: 수집한 데이터를 원래 목적으로만 사용하는가?
□ 보관 기간: "24시간 후 삭제" 같이 정해졌는가?
□ 접근 제어: 누가 데이터에 접근할 수 있는가?
□ 암호화: 전송(HTTPS)과 보관(encryption at rest) 모두 되는가?
□ 규정 준수: GDPR, CCPA, 로컬 개인정보보호법 준수?
□ 제3자 공유: 데이터를 다른 회사와 공유하는가?
□ 사용자 권리: 사용자가 "내 데이터 삭제" 요청할 수 있는가?

위험 사례:
- OpenAI: 사용자 대화가 모델 학습에 사용 → 프라이버시 우려
- 우리: 고객 계약서를 모델 Fine-tuning에 사용? (위험!)
```

<!-- 💬 [논쟁 지점] 한국의 PIPA(개인정보보호법) vs GDPR의 차이
     PIPA: 개인 식별 정보 중심, 기업이 "합리적 수준의 보안" 요구
     GDPR: 데이터 주체의 권리 중심, "개인정보 삭제권" 명시
     실무: 한국 기업도 GDPR 준수 필요 (해외 고객 있으면)
     한국의 AI기본법(2025): "고위험 AI" 분류 시 별도 감시 필요 -->

**Step 3: 투명성(Transparency) 검토**

```
❓ 사용자가 AI의 결정을 이해할 수 있는가?

계층별 투명성:

레벨 1: 기본 정보
- "이것은 AI가 생성한 콘텐츠입니다"
- "신뢰도: 7.2/10"

레벨 2: 의사결정 근거
- "왜 신뢰도가 7.2/10인가?"
- "정확도가 95% 벤치마크에서 이 정도"

레벨 3: 상세 설명
- "이 결정에 영향을 준 요소들"
- 사용자가 이해할 수 있는 언어로

우리의 AI 요약:
✅ 신뢰도 점수 표시: 7.2/10
✅ 이유: "이 영역의 정보가 제한적이어서 신뢰도 낮음"
❓ 개선: "신뢰도가 낮은 부분은 직접 검토하세요" 가이드
```

**Step 4: 책임(Accountability) 명확화**

```
❓ AI가 오류를 낸 경우, 누가 책임지는가?

시나리오:
1. AI가 "계약서에 위험 조항 없음"이라고 했지만, 실제로는 있었다
   → 고객이 손해를 입음
   → 누가 책임지나?

책임 명확화:
┌─────────────────────────────────────┐
│ 제조사 책임 (우리 회사)             │
│ - AI 모델 성능 부족                 │
│ - 대충 출시한 책임                  │
│ → 배상 책임? 부분적 (신뢰도 표시)   │
├─────────────────────────────────────┤
│ 사용자 책임                          │
│ - "신뢰도 낮은 부분은 검토하세요"   │
│   안내를 무시했나?                  │
│ → 배상 책임? 부분적 (과실 상계)     │
├─────────────────────────────────────┤
│ 법적 구조                            │
│ 서비스 약관에 명시:                 │
│ "AI 요약은 참고용이며,             │
│  중요한 결정은 반드시 전문가 검증   │
│  필수"                              │
└─────────────────────────────────────┘

액션:
- 약관에 AI 한계 명시
- "전문가 검증 필수" 강조
- 오류 발생 시 보험 커버 검토
```

**Step 5: 접근성(Accessibility) 검토**

```
❓ 모든 사용자(장애인 포함)가 쓸 수 있는가?

체크:
□ 시각 장애: 스크린 리더로 신뢰도 점수를 읽을 수 있는가?
□ 청각 장애: 음성 알림이 있다면 자막이 있는가?
□ 운동 장애: 마우스 없이 키보드로만 조작 가능한가?
□ 인지 장애: UI가 복잡하지는 않은가?
□ 다국어: 한국어만 지원? (북한 사용자 포함)
□ 저속 네트워크: 느린 인터넷에서도 작동하는가?

우리의 AI 요약 접근성:
✅ WCAG 2.1 AA 준수 (웹 접근성 표준)
✅ 스크린 리더 지원: "신뢰도 7.2 (높음)"으로 읽음
❓ 개선: 모바일 앱에서도 동일 접근성?
```

**Step 6: 노동 윤리(Labor Ethics) 검토**

```
❓ 이 AI가 사람의 일자리를 없애진 않나?

프레임:
AI 도입 = 일자리 제거 (나쁜) vs 역량 강화 (좋은)

우리의 경우:
❌ 나쁜 사례:
"AI 요약으로 법무팀 20명 → 5명으로 감축"
→ 고용 불안정, 사회 신뢰 저하

✅ 좋은 사례:
"AI 요약으로 법무팀이 단순 검토 대신 전략 법무에 집중"
→ 팀의 가치 상승, 급여 상승, 직무 만족도 증가

액션:
- 공개적으로: "이 AI는 일자리를 제거하지 않고 강화합니다"
- 실천적으로: 감축된 인원 없이, 업무 효율화만 추구
- 커뮤니케이션: 고용 공고 시 "AI와의 협력 업무" 강조
```

<!-- 💬 [자기 검증] 한국 기업에서 "일자리 강화"가 실현되는가?
     현실: 많은 기업이 AI를 "비용 절감"으로 해석 → 감축
     해결: 명확한 "역량 강화 전략"을 수립해야 함
     도구: 잡플래닛, 블라인드 등에서 "AI 도입 후 고용" 평판 모니터링
     한국 차별금지법: AI로 인한 차별(예: 특정 직급 우선 감축)도 규제 대상 -->

### AI 윤리 체크리스트 (완전한 버전)

```markdown
# AI 윤리 리뷰 체크리스트: [기능명]

## 1. 편향성 (Bias) 평가

### 1.1 성별 편향
- [ ] 테스트 데이터: 남성 100명, 여성 100명 포함
- [ ] 성능 비교: 남성/여성별 정확도 ≥ 95% 동등?
- [ ] 신뢰도 편차: 남성/여성 신뢰도 차이 < 0.3점?
- [ ] 결과: ✅ 통과 / ⚠️ 주의 / ❌ 실패

### 1.2 나이 편향
- [ ] 20대, 40대, 60대 샘플로 테스트
- [ ] 연령별 성능 동등?
- [ ] 결과: ✅ 통과 / ⚠️ 주의 / ❌ 실패

### 1.3 지역/문화 편향
- [ ] 서울, 지방, 해외 데이터 균형?
- [ ] 지역별 성능 동등?
- [ ] 결과: ✅ 통과 / ⚠️ 주의 / ❌ 실패

### 1.4 학력/사회경제 편향
- [ ] 다양한 배경의 사용자 샘플?
- [ ] 성능 동등?
- [ ] 결과: ✅ 통과 / ⚠️ 주의 / ❌ 실패

**종합 편향성 평가**:
- 🟢 우수: 모든 테스트 통과
- 🟡 주의: 1-2개 항목 주의
- 🔴 실패: 3개 이상 편향성 발견 → 배포 연기

---

## 2. 데이터 프라이버시

### 2.1 데이터 수집 동의
- [ ] 사용자에게 명시적 동의 구함?
- [ ] 동의 텍스트가 명확한가? (법률가 검토)
```

## 🔧 Layer 3: Build & Automate

> **이 레이어는 누구를 위한 건가요?**
> AI 엔지니어, 개발자 — AI 윤리 감사 파이프라인을 자동화합니다.

### 📌 시작 전에

**이 코드가 하는 일:**
Layer 1-2에서 수동으로 했던 과정을 Python 스크립트로 자동화합니다.

**왜 자동화하나?**
AI 모델은 업데이트될 때마다 윤리 검토가 필요합니다. 자동화하면 배포 파이프라인에 통합해서 출시 전 자동 검사가 가능합니다.

**전제조건:**
- Python 3.8+ 설치
- 필요한 패키지 설치 (코드 상단의 import 참고)
- Anthropic API 키 발급 → 환경변수 설정: `export ANTHROPIC_API_KEY=your_key`

> 📖 더 자세한 설정 방법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


### Python + API 자동화

```python
import json
import statistics
from collections import defaultdict
from anthropic import Anthropic

class AIEthicsAuditor:
    """AI 제품의 윤리 감사를 자동화하는 도구"""

    ETHICS_DIMENSIONS = [
        "bias", "privacy", "transparency",
        "accountability", "accessibility", "labor_ethics"
    ]

    SEVERITY = {"🟢": "우수", "🟡": "주의", "🔴": "심각"}

    def __init__(self, feature_name: str, feature_context: dict):
        self.feature_name = feature_name
        self.context = feature_context
        self.client = Anthropic()
        self.results = {}

    def detect_bias(self, test_results: list[dict]) -> dict:
        """
        편향성 감지: 인구통계 그룹 간 성능 차이를 통계적으로 검증

        Args:
            test_results: [{"group": "M", "accuracy": 0.94}, ...]
        """
        groups = defaultdict(list)
        for r in test_results:
            groups[r["group"]].append(r["accuracy"])

        group_means = {g: statistics.mean(accs) for g, accs in groups.items()}
        overall_mean = statistics.mean([a for accs in groups.values() for a in accs])

        # Demographic Parity 검사: 그룹 간 최대 차이
        max_diff = max(group_means.values()) - min(group_means.values())
        threshold = 0.05  # 5% 이상 차이면 편향 의심

        severity = "🟢" if max_diff < threshold else ("🟡" if max_diff < 0.10 else "🔴")

        result = {
            "dimension": "편향성 (Bias)",
            "severity": severity,
            "group_accuracy": group_means,
            "max_difference": round(max_diff, 4),
            "threshold": threshold,
            "demographic_parity": max_diff < threshold,
            "recommendation": (
                "그룹 간 성능 차이가 허용 범위 내입니다."
                if max_diff < threshold
                else f"그룹 간 {max_diff:.1%} 차이 발견. 학습 데이터 재검토 필요."
            )
        }
        self.results["bias"] = result
        return result

    def check_privacy(self, data_flow: dict) -> dict:
        """프라이버시 영향 평가"""
        prompt = f"""
        다음 AI 기능의 프라이버시 영향을 평가하세요.

        기능: {self.feature_name}
        데이터 흐름: {json.dumps(data_flow, ensure_ascii=False)}

        평가 항목:
        1. 수집 데이터 유형 (개인정보 포함 여부)
        2. 데이터 보존 기간 및 삭제 정책
        3. GDPR/개인정보보호법 준수 여부
        4. 제3자 공유 여부
        5. 사용자 동의 메커니즘

        JSON으로 출력: {{
            "severity": "🟢|🟡|🔴",
            "pii_types": [...],
            "retention_policy": "...",
            "gdpr_compliant": true/false,
            "recommendations": [...]
        }}
        """

        message = self.client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=2048,
            messages=[{"role": "user", "content": prompt}]
        )
        result = json.loads(message.content[0].text)
        result["dimension"] = "프라이버시 (Privacy)"
        self.results["privacy"] = result
        return result

    def audit_transparency(self, ui_elements: list[str]) -> dict:
        """투명성 감사: AI 고지, 신뢰도 표시, 설명가능성 검증"""
        checks = {
            "ai_disclosure": any("AI" in e or "인공지능" in e for e in ui_elements),
            "confidence_score": any("신뢰도" in e or "confidence" in e.lower() for e in ui_elements),
            "explanation": any("이유" in e or "근거" in e or "why" in e.lower() for e in ui_elements),
        }
        passed = sum(checks.values())
        severity = "🟢" if passed == 3 else ("🟡" if passed >= 1 else "🔴")

        result = {
            "dimension": "투명성 (Transparency)",
            "severity": severity,
            "checks": checks,
            "score": f"{passed}/3",
            "recommendations": [
                item for item, ok in [
                    ("'이것은 AI가 생성한 콘텐츠입니다' 고지 추가", not checks["ai_disclosure"]),
                    ("신뢰도 점수 표시 추가", not checks["confidence_score"]),
                    ("AI 판단 근거 설명 추가", not checks["explanation"]),
                ] if ok
            ]
        }
        self.results["transparency"] = result
        return result

    def generate_report(self) -> dict:
        """종합 윤리 리포트 생성"""
        severity_order = {"🔴": 0, "🟡": 1, "🟢": 2}
        worst = min(
            self.results.values(),
            key=lambda r: severity_order.get(r.get("severity", "🟡"), 1)
        )

        overall = worst["severity"]
        deployable = overall != "🔴"

        return {
            "feature": self.feature_name,
            "overall_severity": overall,
            "deployable": deployable,
            "verdict": (
                "✅ 배포 가능" if overall == "🟢"
                else "⚠️ 조건부 배포 (개선 필요)" if overall == "🟡"
                else "🚫 배포 중단 — 심각한 윤리 이슈"
            ),
            "dimensions": self.results,
            "action_items": [
                r.get("recommendations", [])
                for r in self.results.values()
                if r.get("severity") != "🟢"
            ]
        }


# === 실행 예시 ===
if __name__ == "__main__":
    auditor = AIEthicsAuditor(
        feature_name="AI 문서 요약",
        feature_context={"model": "Claude API", "domain": "법률 문서"}
    )

    # 1. 편향성 검사
    bias = auditor.detect_bias([
        {"group": "M", "accuracy": 0.948},
        {"group": "F", "accuracy": 0.949},
        {"group": "20s", "accuracy": 0.942},
        {"group": "40s", "accuracy": 0.945},
        {"group": "60s", "accuracy": 0.938},
    ])
    print(f"편향성: {bias['severity']} (그룹 간 차이: {bias['max_difference']:.1%})")

    # 2. 투명성 감사
    trans = auditor.audit_transparency([
        "이것은 AI가 생성한 요약입니다",
        "신뢰도: 7.2/10",
        "이 신뢰도의 이유: 법률 용어 밀도가 높아 정확도 제한"
    ])
    print(f"투명성: {trans['severity']} ({trans['score']})")

    # 3. 종합 리포트
    report = auditor.generate_report()
    print(f"\n종합: {report['verdict']}")
    print(json.dumps(report, ensure_ascii=False, indent=2))
```

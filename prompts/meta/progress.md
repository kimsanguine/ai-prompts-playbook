## 🚀 GTM 전략: LinkedIn 5일 시리즈 (다음 단계)

> **목표**: GitHub 퍼블리싱 후 5일 연속 LinkedIn 포스트로 초기 트래픽 확보
> **타겟**: 한국 PM/CPO (1차), AI 관심 개발자 (2차)

### Day 1: 문제 제기 — "PM이 ChatGPT를 잘못 쓰는 3가지 이유"

**핵심 메시지**: AI를 "만능 답변기"로 쓰면 뻔한 결과만 나온다
**구조**:
- Hook: "경쟁사 분석해줘"라고 쳤을 때 vs 구조화된 프롬프트를 썼을 때 결과 비교
- 3가지 실수: ① 역할 없이 질문 ② 추론 전략 없이 지시 ③ 결과 검증 안 함
- CTA: "33개 프롬프트로 이 3가지를 해결한 오픈소스를 만들었습니다" → GitHub 링크
**톤**: 교육적이되 겸손하게. "나도 이런 실수를 했다"

### Day 2: 프레임워크 소개 — "프롬프트에 '레이어'가 있다고?"

**핵심 메시지**: 프롬프트는 난이도가 아니라 사고의 깊이
**구조**:
- Hook: "Layer 1은 5분, Layer 3은 Python — 같은 프롬프트인데 결과가 완전히 다릅니다"
- 3-레이어 설명: 전문가의 눈 → 합의 너머 → 자기 검증
- Harvard 실험 인용: 구조적 질문 = 2x 학습 성과
- CTA: "세 개의 질문" 철학 문서 링크 → GitHub
**톤**: 지적 호기심 자극. "왜 이렇게 설계했는지"

### Day 3: 한국 특화 — "한국 PM이 글로벌 프롬프트를 쓰면 안 되는 이유"

**핵심 메시지**: 영어 프롬프트 번역으로는 한국 비즈니스 맥락을 못 담는다
**구조**:
- Hook: "임원 보고에서 두괄식을 안 쓰면 읽히지도 않습니다"
- 3가지 차이: ① 규제 (PIPA, AI 기본법) ② 문화 (합의제, 품의서) ③ 데이터 (DART, 잡플래닛)
- 예시: 글로벌 PRD 프롬프트 vs 한국 규제 포함 PRD 프롬프트 비교
- CTA: "한국 시장 데이터가 들어간 33개 프롬프트" → GitHub 링크
**톤**: 실무 공감대. "아, 이거 맞아" 반응 유도

### Day 4: 워크플로우 체이닝 — "30개 프롬프트가 연결되는 워크플로우"

**핵심 메시지**: 프롬프트 하나로 끝나는 게 아니라 체인으로 연결될 때 진가 발휘
**구조**:
- Hook: "경쟁사 분석 → 문제 정의 → RICE 우선순위 → PRD → 메트릭스 → 임원 보고가 6단계로 연결됩니다"
- 워크플로우 다이어그램 (이미지 or 텍스트)
- 실제 체이닝 결과 Before/After
- CTA: "📥 워크플로우 입력 섹션을 따라가면 자동으로 연결됩니다" → GitHub 링크
**톤**: "와, 이렇게 되는구나" 놀라움. 실용적 감탄

### Day 5: 오픈소스 선언 — "오픈소스로 공개합니다"

**핵심 메시지**: 카테고리를 만드는 사람이 카테고리를 소유한다
**구조**:
- Hook: "한국 PM을 위한 구조적 AI 프롬프트 플레이북이라는 카테고리는 존재하지 않았습니다"
- 왜 오픈소스인가: PM 커뮤니티 기여 + 함께 만들어가는 프레임워크
- 프로젝트 숫자: 33개 프롬프트, 24,300+ 라인, 26개 Python 클래스, 5개 카테고리
- CTA: ⭐ Star + Fork + 새 프롬프트 기여 요청 → GitHub 링크
**톤**: 진정성 + 커뮤니티 초대. "같이 만들어요"

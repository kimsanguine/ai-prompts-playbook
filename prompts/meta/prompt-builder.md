# 🏗️ 프롬프트 빌더 (AI Prompt Builder for PMs)

> 예상 소요: 10분 (Quick) / 30분 (Deep) / 60분+ (Build)
> 워크플로우: → 모든 프롬프트의 기초

---

## 🎯 이 프롬프트가 해결하는 문제

좋은 AI 결과를 원한다면, 먼저 **좋은 프롬프트를 써야 한다.** 이 프롬프트는 "PM이 좋은 프롬프트를 만드는 방법"을 설명하고, AI 모델(예: Claude)에 요청할 때 사용할 수 있는 메타 프롬프트(프롬프트를 만드는 프롬프트)를 제공한다.

---

## ⚡ Layer 1: Quick Start

> **이 레이어는 누구를 위한 건가요?**
> AI를 처음 쓰는 분, 비개발자, 학생 — 5분이면 나만의 커스텀 프롬프트를 만들 수 있습니다.

### 📌 사용법

1. **ChatGPT, Claude, Gemini 중 아무 AI 챗봇**을 열고 새 대화를 시작하세요
2. 아래 프롬프트를 **통째로 복사**해서 AI에 붙여넣으세요
3. `{중괄호}` 안의 내용만 여러분의 상황에 맞게 바꾸세요
4. Enter를 누르면 끝!

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


```
당신은 프롬프트 아키텍트다.

좋은 프롬프트의 4가지 필수 요소:

1. 역할 (Role): "넌 뭐야?"
   "당신은 AI PM 메트릭 전문가입니다"

2. 목표 (Goal): "뭘 원해?"
   "AI 채택 메트릭을 정의해주세요"

3. 제약 (Constraints): "한계는?"
   "측정 가능해야 하고, 3개 이상 5개 이하"

4. 예상 출력 (Output Format): "결과 형식은?"
   "각 메트릭별: 정의 + 측정방법 + 해석"

추가: 예시 (Examples) - 실제 사례로 보여주기

{역할} + {목표} + {제약} + {출력형식} = 좋은 프롬프트
```

💡 **Quick Tips:**
- 역할부터 명시 (AI가 "어떤 관점"에서 답할지 정하기)
- 예시 1-2개 추가하면 정확도 3배 ↑
- 너무 길지 않되, 모호하지도 않게 (200-500 단어 추천)


### ⚠️ 이런 경우는요?

| 상황 | 대처법 |
|------|--------|
| 어떤 프롬프트를 만들지 모르겠음 | "내가 매주 반복하는 작업"을 리스트업 → 자동화할 수 있는 것부터 |
| 만든 프롬프트가 결과가 안 좋음 | prompt-anatomy.md로 역공학 분석 → 문제점 파악 |
| Layer 3까지 만들어야 하나 | Layer 1만으로도 충분할 수 있습니다. 필요에 따라 레이어 추가 |
| 팀용 프롬프트 만들기 | 변수({})를 명확히 하고, 예시를 반드시 포함하세요 |

## 🧠 Layer 2: Deep Dive

> **이 레이어는 누구를 위한 건가요?**
> PM, 기획자, 마케터 — Playbook의 패턴을 활용해 완전히 새로운 프롬프트를 만들고 싶은 분입니다.
"내 업무에 맞는 프롬프트가 없어서 직접 만들고 싶다"면 이걸 써보세요.

### 📌 진행 방식

이 프롬프트는 **5단계 CoT(Chain of Thought)** 구조입니다. 두 가지 방식으로 쓸 수 있습니다:

**방식 A: 한 번에 넣기 (추천)**
- 아래 전체를 통째로 복사해서 AI에 붙여넣으세요
- AI가 모든 단계를 순서대로 실행합니다
- 결과가 한 번에 나옵니다 (길지만 완결적)

**방식 B: 단계별로 넣기 (더 깊은 분석)**
- 1-2단계를 먼저 넣고 결과를 확인하세요
- 결과가 만족스러우면 다음 단계를 이어서 넣으세요
- 중간에 수정이 필요하면 해당 단계를 수정 요청할 수 있습니다
- 시간이 더 걸리지만, 각 단계를 직접 검증할 수 있습니다

**`<!-- 💬 -->` 교육 주석이란?**
- 프롬프트 안에 `<!-- 💬 ... -->` 형태의 주석이 있습니다
- 이건 **여러분을 위한 설명**이지, AI에게 보내는 지시가 아닙니다
- AI에 붙여넣을 때 포함해도 되고 빼도 됩니다 (결과에 영향 없음)
- "왜 이렇게 설계했는가?"를 이해하면 더 좋은 프롬프트를 만들 수 있습니다

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


### 역할 설정

당신은 **프롬프트 엔지니어(Prompt Engineer)**이다. 당신의 역할:

1. **좋은 프롬프트 설계**: 명확한 구조 + 예시
2. **AI 특성 이해**: 모델 한계와 강점을 알고 활용
3. **반복 개선**: "첫 결과 → 조정 → 더 나은 결과"

### 목표

최종 아웃풋:
- **프롬프트 템플릿** (자주 쓰는 패턴)
- **프롬프트 체크리스트** (빠뜨린 게 있나?)
- **반복 개선 가이드** (결과가 나쁘면 어떻게?)
- **AI 모델별 최적화** (Claude vs GPT-4)

### Reasoning Strategy

<!-- 💬 좋은 프롬프트와 나쁜 프롬프트의 차이는?

나쁜 프롬프트:
"요약해줘"
→ 뭘 요약해? 어떤 스타일로? 얼마나 길게?
→ 결과가 예측 불가능

좋은 프롬프트:
"당신은 법무팀 담당자입니다. 이 계약서를 다음 형식으로 요약:
 1. 당사자 정보
 2. 주요 의무
 3. 위험 조항 3개
 4. 권장 액션
 각 섹션은 3-5줄로, 법적 용어는 쉬운 말로 변환하세요."
→ 예측 가능한, 구조화된 결과
-->

**Step 1: 역할(Role) 명확히**

```
좋은 역할 정의의 3가지 요소:

1. 전문 영역
   "당신은 AI 제품 메트릭 전문가입니다"
   vs
   "당신은 데이터 분석가입니다"
   → 전자가 더 구체적, 맥락 제공

2. 배경 & 경험
   "15년 AI 제품 경험을 가진 Senior PM"
   vs
   "최신 AI 트렌드에 밝은 전문가"
   → 경험이 있으면 더 신뢰도 높은 답변

3. 성향/가치관
   "당신은 데이터 기반 의사결정을 중시합니다"
   → 모델의 "관점" 설정
```

**Step 2: 목표(Goal) 구체화**

```
❌ 모호한 목표:
"메트릭을 만들어줄 수 있을까?"

✅ 구체적 목표:
"AI 채택 메트릭 프레임워크를 정의해주세요.
 조건:
 - 측정 가능해야 함 (쿼리/설문으로 계산 가능)
 - 비즈니스 임팩트와 연결 (ROI 측정 가능)
 - 3-5개 메트릭만 (너무 많으면 관리 어려움)
 결과물: 각 메트릭별 정의 + SQL 쿼리 + 해석"
```

**Step 3: 제약(Constraints) 설정**

```
제약의 종류:

1. 범위 제약
   "한국 SaaS 제품만 고려"
   "엔터프라이즈 시장만 (SMB 제외)"

2. 길이 제약
   "500단어 이하"
   "한 페이지 분량"

3. 형식 제약
   "마크다운 표 형식으로"
   "코드 포함"

4. 콘텐츠 제약
   "기술 용어 최소화 (비기술 독자용)"
   "구체적 사례 2개 이상 포함"

5. 스타일 제약
   "한국식 비즈니스 관례 반영"
   "실행 중심 (이론 아님)"
```

**Step 4: 출력 형식(Output Format) 명시**

```
명확한 출력 형식 = 더 나은 결과

❌ 모호함:
"메트릭 정의해줘"

✅ 명확함:
"마크다운 표 형식으로 다음을 포함해주세요:
| 메트릭명 | 정의 | 측정방법 | 목표치 | 해석 |
|---------|------|---------|--------|------|
| 예) ... | ... | ... | ... | ... |"

또는:
"JSON 형식으로 다음 스키마로:
{
  'metric_name': 'string',
  'definition': 'string (50단어 이내)',
  'measurement': {...},
  'target': {...},
  'interpretation': 'string'
}"
```

**Step 5: 예시(Examples) 추가**

```
❌ 예시 없음:
"AI 메트릭을 정의해주세요"

✅ 예시 포함:
"예를 들어, 'Activation Rate'라면:
 정의: 가입한 사용자 중 AI 기능을 처음 사용한 비율
 측정: (첫 AI 사용자 / 전체 신규 사용자) × 100
 목표: 25% 이상
 해석: 25% 이상이면 발견도 좋음, 이하면 UX 문제

이런 식으로 3-5개 메트릭을 정의해주세요."

→ AI가 패턴을 이해하고 같은 수준의 답변 생성
```

### 프롬프트 체크리스트

프롬프트를 쓴 후, 다음을 확인:

```
□ 역할이 명확한가?
  "금융 분야 10년 경력의 PM"처럼 구체적인가?

□ 목표가 명확한가?
  "뭘 원하는가?"를 한 문장으로 설명 가능한가?

□ 제약이 명시되어 있는가?
  범위, 길이, 형식, 콘텐츠, 스타일

□ 출력 형식이 명확한가?
  AI가 "어떤 형식으로 돌려주길 원하는가?"

□ 예시가 있는가?
  1-2개의 구체적 예시로 패턴 전달

□ 어조/톤이 명확한가?
  "전문적으로", "쉽게", "유머있게" 등

□ 한국 문화/관례가 반영되어 있는가?
  (해당되는 경우)

□ 길이가 적당한가?
  너무 길면 (500단어 이상) 요약, 너무 짧으면 예시 추가
```

### 프롬프트 템플릿 (자주 쓰는 패턴)

```markdown
# 프롬프트 템플릿 #1: 분석 & 리포트

당신은 {분야} 전문가입니다. {배경/경험}.

{주제}에 대해 다음을 분석해주세요:

목표:
{명확한 목표, 2-3 문장}

요구사항:
- {제약 1}
- {제약 2}
- {제약 3}

출력 형식:
{구체적 형식, 가능하면 예시}

예시:
{1-2개의 구체적 예시}

특별히 고려할 점:
- {한국 문화/관례}
- {산업별 고유 특성}
- {금기사항/피해야 할 것}
```

```markdown
# 프롬프트 템플릿 #2: 코딩/SQL

당신은 {언어} 전문 데이터 엔지니어입니다.

다음 {요청}을 구현해주세요:

요구사항:
- {기능 1}
- {기능 2}
- {성능 요구사항}

입력:
{입력 데이터 정보}

출력:
{출력 형식}

예시:
입력: {구체적 입력 예시}
→ 기대 출력: {구체적 출력}

주의사항:
- {성능 최적화 팁}
- {보안 고려사항}
```

```markdown
# 프롬프트 템플릿 #3: 창의적 콘텐츠

당신은 {분야} 콘텐츠 크리에이터입니다. {스타일/톤}.

{대상 청중}을 위해 다음을 작성해주세요:

주제:
{주제, 핵심 메시지}

형식:
{구체적 형식, 길이}

톤:
{프로페셔널/캐주얼/유머러스 등}

포함해야 할 것:
- {핵심 포인트 1}
- {핵심 포인트 2}

피해야 할 것:
- {피해야 할 표현/관점}

예시:
[좋은 예시 1]
[좋은 예시 2]
```

### 프롬프트 반복 개선 가이드

결과가 원하는 것과 다르면:

```
❌ "다시 해줄 수 있어?" (모호함)

✅ "다시 시도해주세요. 구체적으로:
   - {예상했던 점}: 이렇게 나왔어요
   - {실제 결과}: 이렇게 나왔어요
   - {차이}: {차이점을 구체적으로 설명}

   조정 사항:
   1. {역할 조정} (예: "더 기술적이어야 함")
   2. {제약 조정} (예: "한국 예시 더 추가")
   3. {포맷 조정} (예: "테이블 형식으로")

   이번에는 이 점들을 반영해주세요."

→ AI가 "무엇을 잘못했는가"를 이해하고 개선
```

### AI 모델별 최적화

```
Claude (Anthropic):
✅ 긴 컨텍스트 (100K tokens)
✅ 깊은 사고 (Thinking feature)
✅ 한국어 이해도 높음
권장: 복잡한 분석, 한국 특화 요청

GPT-4 (OpenAI):
✅ 코딩 성능 우수
✅ 도구 활용 (Code Execution)
권장: 프로그래밍, 실시간 검색 필요한 경우

Gemini (Google):
✅ 멀티모달 (이미지, 영상)
권장: 비전 태스크, 이미지 분석

Llama (Meta):
✅ 오픈소스, 빠름
권장: 내부 배포, 비용 최적화
```

💡 **Deep Tips:**
- **역할이 핵심**: 좋은 역할 정의 = 절반 이상의 성공
- **예시의 힘**: 예시 1-2개 추가하면 정확도 2-3배 ↑
- **피드백 루프**: 처음에 100% 완벽할 리 없으니, 반복 개선 기준 설정


### ⚠️ 단계별 예외 처리

| 단계 | 예외 상황 | 대처법 |
|------|-----------|--------|
| 1단계 | 목표 정의가 모호 | "이 프롬프트의 결과물로 누가 무엇을 할 것인가?"로 명확히 |
| 2단계 | 구조 설계 어려움 | 기존 Playbook 프롬프트의 구조를 참고하세요 |
| 3단계 | 변수 설계 어려움 | "사용자가 매번 바꿔야 하는 부분"을 {중괄호}로 |
| 4단계 | 테스트가 어려움 | 3가지 다른 시나리오로 테스트해보세요 |

## 🔧 Layer 3: Build & Automate

> **이 레이어는 누구를 위한 건가요?**
> AI 엔지니어, 개발자 — 프롬프트 생성 파이프라인을 자동화합니다.

### 📌 시작 전에

**이 코드가 하는 일:**
Layer 1-2에서 수동으로 했던 과정을 Python 스크립트로 자동화합니다.

**왜 자동화하나?**
팀 내에서 여러 프롬프트를 만들 때 일관된 품질을 유지하기 위해 자동화합니다.

**전제조건:**
- Python 3.8+ 설치
- 필요한 패키지 설치 (코드 상단의 import 참고)

### Prompt Optimization Framework (Python)

```python
# prompt_optimizer.py (개요)

class PromptAnalyzer:
    """프롬프트 품질 분석 & 개선"""

    def analyze(self, prompt: str) -> dict:
        """프롬프트를 5개 항목으로 평가"""
        return {
            'role_clarity': self.score_role(prompt),        # 0-10
            'goal_clarity': self.score_goal(prompt),        # 0-10
            'constraints': self.score_constraints(prompt),  # 0-10
            'output_format': self.score_format(prompt),     # 0-10
            'examples': self.score_examples(prompt),        # 0-10
            'overall': self.calculate_overall_score(),      # 0-10
        }

    def suggest_improvements(self, analysis: dict) -> List[str]:
        """약한 부분 개선 제안"""
        suggestions = []
        if analysis['role_clarity'] < 7:
            suggestions.append("역할을 더 구체적으로 (경험, 전문성)")
        if analysis['examples'] == 0:
            suggestions.append("구체적 예시 1-2개 추가")
        return suggestions

    def optimize(self, prompt: str) -> str:
        """프롬프트 자동 개선"""
        return f"""원래 프롬프트:
{prompt}

개선된 프롬프트:
[분석 기반 자동 개선]"""

class PromptBuilder:
    """프롬프트 빌더"""

    def build_from_template(self,
                           template: str,
                           **kwargs) -> str:
        """템플릿으로부터 프롬프트 생성"""
        return template.format(**kwargs)

    def validate(self, prompt: str) -> bool:
        """필수 요소 확인"""
        required = ['role', 'goal', 'constraints', 'format']
        return all(elem in prompt.lower() for elem in required)
```

### 프롬프트 품질 분석

```
원본 프롬프트:
"AI 메트릭을 정의해주세요"

분석:
- 역할 명확도: ❌ 0/10 (역할 없음)
- 목표 명확도: ⚠️ 3/10 (모호함)
- 제약: ❌ 0/10 (없음)
- 출력 형식: ❌ 0/10 (없음)
- 예시: ❌ 0/10 (없음)
전체 점수: 0.6/10

개선 제안:
1. "당신은 AI PM 메트릭 전문가입니다"로 시작
2. "AI 채택을 측정하는 3-5개 메트릭을 정의하세요" (구체화)
3. 측정 가능성, 한국 시장 고려 명시
4. 마크다운 표 형식 지정
5. 예시 메트릭 1-2개 제시

개선된 프롬프트:
당신은 AI 제품 메트릭 전문가입니다.
AI 채택을 측정하는 핵심 메트릭 프레임워크를 정의해주세요.

요구사항:
- 측정 가능해야 함 (데이터로 계산 가능)
- 비즈니스 임팩트와 연결 (ROI 측정)
- 3-5개만 (관리 가능)
- 한국 SaaS 제품 기준

출력: 마크다운 표
| 메트릭 | 정의 | 측정 | 목표 |

예시:
Activation Rate: 가입 사용자 중 첫 AI 사용 비율...

개선된 점수: 8.5/10 ✅
```

### 템플릿 기반 프롬프트 생성

```
템플릿:
당신은 {분야} 전문가입니다.

형식: {형식}
제약: {제약들}

입력:
- 분야: "AI 제품"
- 대상: "경영진"
- 주제: "메트릭 프레임워크"
- 형식: "Executive Summary + 상세 표"
- 제약: "3분 읽을 수 있는 분량, 한국 예시 포함"

생성된 프롬프트:
당신은 AI 제품 전문가입니다.
CFO를 위해 AI 채택 메트릭 프레임워크를 분석해주세요.

형식:
1. Executive Summary (1분)
2. 핵심 3개 메트릭 표 (1분)
3. 각 메트릭의 비즈니스 임팩트 (1분)

제약:
- 총 3분 읽을 수 있는 분량
- 한국 금융/보험사 사례 포함
- 금액(ROI) 명시
```

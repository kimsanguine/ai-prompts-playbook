## Q2. GitHub + LinkedIn 홍보 시 사용자 호응 예측

### 긍정 신호 (호응할 근거)

1. **블루오션 확인**: 한국어 PM 특화 프롬프트 리포 = 경쟁자 0
2. **LinkedIn 파워**: LinkedIn이 AI 검색에서 전문 콘텐츠 최다 인용 도메인 (#1, 2025-2026 데이터)
3. **커뮤니티 수요**: 지피터스/디스콰이엇에서 "PM을 위한 AI 활용법" 콘텐츠가 높은 인게이지먼트
4. **차별화 명확**: 3-레이어 인지 구조 + 한국 시장 데이터 = 어디에도 없는 조합
5. **GitHub Star 선례**: deanpeters 리포가 단순 프롬프트 나열로 321★ → 우리는 훨씬 깊이 있음

### 리스크 요인 (호응이 약할 수 있는 부분)

1. **진입 장벽**: 33개 프롬프트 × 3레이어 = 학습 곡선이 높아 보일 수 있음
2. **즉시성 부족**: 프롬프트 마켓플레이스는 "복사 → 붙여넣기"인데, 우리는 "읽고 → 이해하고 → 적용"
3. **타겟 좁음**: PM/CPO만 타겟 → 범용 프롬프트 대비 도달 범위 작음
4. **GitHub 한국어**: 한국어 중심 리포는 글로벌 Star를 못 받음 → 영문 README 필요

### 호응 예측: ⭐ 중상 (조건부 열광)

> **"한국 PM 커뮤니티에서는 열광할 가능성 높음 (특히 AI SaaS PM)"**
> 단, "처음 5분 경험"이 핵심. README 랜딩 + 퀵스타트가 약하면 이탈.

## Q3. 경쟁 대비 아쉬운 점 (솔직한 약점 분석)

### 🔴 크리티컬 (반드시 보완)

| 약점 | 설명 | 경쟁자 대비 |
|------|------|-----------|
| **인터랙티브 없음** | GitHub md 파일 = 정적 텍스트. 즉시 실행/테스트 불가 | 오픈프롬프트는 즉시 실행 가능, Aakash Gupta는 Notion 템플릿 |
| **온보딩 경로 부재** | 33개 파일 중 "나한테 맞는 프롬프트"를 찾기 어려움 | dair-ai는 주제별 네비게이션이 직관적 |
| **영문 버전 없음** | 글로벌 Star 획득 불가 → 신뢰도 신호 약함 | deanpeters 321★도 영문이라 글로벌 도달 |

### 🟡 중요 (2주 내 보완 권장)

| 약점 | 설명 |
|------|------|
| **실전 사례 부족** | "이 프롬프트로 실제 뭘 만들었는지" 보여주는 Before/After 없음 |
| **커뮤니티 전략 부재** | GitHub Issue/Discussion 활용 계획 없음 |
| **기여 가이드 없음** | CONTRIBUTING.md 없으면 오픈소스 커뮤니티 성장 불가 |
| **비주얼 부족** | README에 다이어그램/스크린샷 없으면 스크롤 탈출 |

### 🟢 나이스투해브

| 약점 | 설명 |
|------|------|
| **Notion/Obsidian 템플릿** | 파워유저는 GitHub md보다 Notion 템플릿을 원함 |
| **실행 결과 갤러리** | "이 프롬프트의 출력물 예시" 페이지 |
| **버전별 변경 로그** | 이미 progress.md 있지만 CHANGELOG.md 형식이 표준 |

## Q4. 집중해야 할 포인트 — 고객 호응 극대화 전략

### 🎯 Phase 1: 런칭 전 필수 (1주)

#### 1. "5분 안에 가치 증명" README 리디자인
```
현재: 철학 → 구조 설명 → 파일 목록
개선:
  Hook (PM이 겪는 문제)
  → 30초 퀵스타트 (가장 인기 프롬프트 1개 즉시 복사)
  → Before/After 스크린샷
  → 전체 구조 설명
```

#### 2. "나에게 맞는 프롬프트 찾기" 가이드
```
Q: 당신의 역할은?
  → PM → {5개 추천 프롬프트}
  → CPO → {3개 추천 프롬프트}
  → 디자이너 → {3개 추천 프롬프트}
  → 엔지니어 → {3개 추천 프롬프트}

Q: 지금 가장 급한 일은?
  → 스프린트 플래닝 → sprint-kickoff.md + backlog-prioritization.md
  → PRD 작성 → requirements-to-prd.md + feature-spec.md
  → 경쟁사 분석 → company-profiling.md + positioning-strategy.md
```

#### 3. 영문 README.md (최소 버전)
- 제목 + 한줄 설명 + 구조 다이어그램만 영문
- "Korean-first PM Prompt Playbook with 3-layer cognitive structure"
- 글로벌 Star 유입 경로 확보

### 🎯 Phase 2: 런칭 주간 LinkedIn 전략

#### 콘텐츠 시리즈 (5일)
```
Day 1: "PM이 ChatGPT를 잘못 쓰는 3가지 이유" (문제 제기 → 저장소 링크)
Day 2: "프롬프트에 '레이어'가 있다고?" (세 질문 프레임워크 소개)
Day 3: "한국 PM이 글로벌 프롬프트를 쓰면 안 되는 이유" (한국 특화 내용 하이라이트)
Day 4: "30개 프롬프트가 연결되는 워크플로우" (체이닝 데모)
Day 5: "오픈소스로 공개합니다" (GitHub 링크 + Star 요청)
```

#### 타겟 커뮤니티 시딩
```
1순위: 디스콰이엇 (72만 IT/PM) → 프로덕트 카테고리 게시
2순위: 지피터스 (AI 커뮤니티) → "PM을 위한 AI" 프레임
3순위: 퍼블리/브런치 → 롱폼 아티클로 깊이 전달
4순위: 오픈카톡방 (PM 채널) → 직접 공유
```

### 🎯 Phase 3: 성장 (런칭 후 1개월)

#### 피드백 루프 구축
```
GitHub Discussions 활성화:
  - "이 프롬프트로 만든 결과물 공유" 스레드
  - "한국 시장 데이터 제보" 스레드
  - "새 프롬프트 요청" 스레드

CONTRIBUTING.md 작성:
  - 프롬프트 작성 가이드 (레이어 구조 준수)
  - 한국 시장 데이터 기여 가이드
  - PR 리뷰 기준
```

#### 확장 방향
```
1. Notion 템플릿 패키지 (높은 수요 예상)
2. 주간 "프롬프트 오브 더 위크" 뉴스레터
3. AI 프롬프트 워크숍 (디스콰이엇/지피터스 콜라보)
```

# 📖 프롬프트 사용 가이드

> 이 문서는 AI Prompts Playbook의 모든 프롬프트를 **처음 쓰는 분**을 위한 공통 안내서입니다.
> 각 프롬프트 파일에도 해당 프롬프트에 특화된 사용법이 있지만, 여기서 전체적인 흐름을 먼저 이해하세요.

---

## 먼저 읽어야 할 것

이 Playbook의 레이어는 **난이도가 아니라 질문의 구조**입니다.

| 레이어 | 던지는 질문 | 빌려오는 것 |
|--------|-----------|-----------|
| ⚡ Layer 1 | "전문가는 이걸 어떤 틀로 보나요?" | 전문가의 프레이밍 |
| 🧠 Layer 2 | "전문가들도 답을 못 내린 부분은?" | 확실한 것/불확실한 것의 경계 |
| 🔧 Layer 3 | "내가 이해한 걸 어떻게 테스트하나?" | 검증 가능한 코드 |

자세한 철학은 [세 개의 질문 — AI 시대의 사고법](three-questions.md)을 참고하세요.

---

## 프롬프트 파일 구조

모든 프롬프트는 이렇게 생겼습니다:

```
🎯 이 프롬프트가 해결하는 문제  ← "왜 필요한가?"
📥 이전 프롬프트에서 받는 입력  ← 워크플로우 체인 연결
⚡ Layer 1: 전문가의 눈 빌리기  ← 5분이면 전문가의 프레이밍을 빌림
🧠 Layer 2: 합의 너머 보기     ← CoT + 교육 주석으로 경계 탐색
🔧 Layer 3: 자기 검증          ← 코드로 가설을 데이터에 부딪혀봄
📊 실행 결과 예시               ← "이런 결과가 나옵니다"
🔍 이 결과를 받은 후            ← 전문가라면 던질 후속 질문
🔗 다음 단계                    ← 연결 프롬프트
```

**꼭 Layer 1부터 시작할 필요는 없습니다.** 하지만 세 질문을 모두 던져야 한 분야의 뼈대가 완성됩니다.

---

## ⚡ Layer 1: 전문가의 눈 빌리기

### 이 레이어가 하는 일
전문가가 이 분야를 볼 때 쓰는 **프레이밍(해석 틀)**을 빌려줍니다. 같은 데이터를 봐도 읽어내는 것이 다른 이유는 지식의 양이 아니라 해석하는 틀의 차이입니다. Layer 1은 이 틀을 5분 만에 세워줍니다.

### 대상
AI를 처음 쓰는 분, 비개발자, 학생, "일단 빠르게 결과를 보고 싶은" 모든 분

### 사용할 수 있는 AI 도구
어떤 AI 챗봇이든 사용할 수 있습니다:

| AI 도구 | 접속 방법 | 무료 사용 | 특징 |
|---------|----------|-----------|------|
| **ChatGPT** | [chat.openai.com](https://chat.openai.com) | 무료 (GPT-3.5) / 유료 (GPT-4) | 가장 많이 쓰임 |
| **Claude** | [claude.ai](https://claude.ai) | 무료 (일부 제한) / 유료 | 긴 텍스트 분석에 강함 |
| **Gemini** | [gemini.google.com](https://gemini.google.com) | 무료 | Google 생태계 연동 |

### 단계별 실행

**1단계: AI 도구를 열고 "새 대화"를 시작하세요**
- ChatGPT: 왼쪽 상단 "New chat" 클릭
- Claude: 왼쪽 상단 "New chat" 클릭
- Gemini: 왼쪽 메뉴에서 "New chat"

**2단계: 프롬프트의 `⚡ Layer 1` 코드 블록을 통째로 복사하세요**
- 코드 블록(` ``` `)으로 감싸진 텍스트를 전부 복사합니다
- 코드 블록 오른쪽 상단의 📋 복사 버튼을 쓰면 편합니다

**3단계: `{중괄호}` 안의 내용을 여러분의 상황에 맞게 바꾸세요**

예시:
```
Before: {경쟁사명}의 사용자 리뷰 감성을 분석하세요.
After:  Notion의 사용자 리뷰 감성을 분석하세요.

Before: {AI 기능명}에서 현재 상태: {메트릭}
After:  AI 번역 기능에서 현재 상태: 정확도 85%
```

**4단계: AI 챗봇에 붙여넣고 Enter**
- 결과가 길게 나올 수 있습니다 (정상)
- 결과가 잘리면 "계속 작성해주세요"라고 입력하세요

### 자주 하는 실수

| 실수 | 올바른 방법 |
|------|------------|
| `{중괄호}`를 그대로 둠 | 반드시 자기 상황에 맞는 내용으로 바꾸세요 |
| 프롬프트 일부만 복사 | 코드 블록 전체를 복사하세요. 부분만 넣으면 맥락이 끊깁니다 |
| 결과가 마음에 안 들어서 포기 | "더 구체적으로 분석해주세요" 또는 "표 형식으로 정리해주세요"라고 추가 요청하세요 |
| 데이터 없이 분석 요청 | 분석 프롬프트는 데이터가 필요합니다. 리뷰, CSV 등을 함께 붙여넣으세요 |

### 결과가 기대보다 낮을 때

- **"더 구체적으로"**: "각 항목에 수치와 근거를 포함해서 다시 작성해주세요"
- **"형식 변경"**: "위 내용을 표 형식으로 정리해주세요"
- **"깊이 추가"**: "각 항목에 '왜?'를 추가해서 원인 분석을 포함하세요"
- **"한국 맥락"**: "한국 시장/한국 사용자 관점에서 다시 해석해주세요"

→ 이런 후속 질문이 반복된다면, **Layer 2로 올라갈 때**입니다.

---

## 🧠 Layer 2: 합의된 영역 너머 보기

### 이 레이어가 하는 일
교과서에 나오는 내용은 어디서든 찾을 수 있습니다. Layer 2는 **전문가들조차 답을 못 내린 지점**을 탐색합니다. 확실한 것과 불확실한 것을 구분할 수 있어야 그 분야를 안다고 할 수 있습니다.

### 대상
PM, 기획자, 마케터, 컨설턴트 — "AI에게 더 깊이 생각하게 만들고 싶은" 분

### Layer 2의 핵심 개념: Reasoning Strategy

Layer 1이 "전문가의 틀 빌리기"라면, Layer 2는 **"그 틀의 경계를 탐색하기"**입니다.

```
Layer 1: "경쟁사 분석해줘" → 표면적 분석

Layer 2: "1단계: 데이터 수집 → 2단계: 패턴 분석 → 3단계: 기회 도출 → 4단계: 자기검증"
         → 각 단계의 output이 다음 단계의 input이 되어 점점 깊어짐
```

이것을 **CoT (Chain of Thought)**라고 하며, AI의 분석 품질을 30-50% 높이는 검증된 기법입니다.

### 단계별 실행

**방식 A: 한 번에 넣기 (추천, 시간 절약)**

1. Layer 2의 프롬프트 전체를 복사합니다
2. `{중괄호}` 변수를 채웁니다
3. AI에 붙여넣고 Enter
4. AI가 모든 단계를 순서대로 실행합니다

**방식 B: 단계별로 넣기 (더 정밀한 통제)**

1. 1-2단계만 먼저 복사해서 AI에 넣습니다
2. 결과를 확인합니다 — 만족스러운가?
3. 만족스럽다면 "이어서 3-4단계를 실행하세요"
4. 중간에 수정이 필요하면: "2단계에서 X를 Y로 바꾸고 다시 분석하세요"
5. 마지막 단계(자기검증)까지 완료

**방식 B를 쓰면 좋은 경우:**
- 데이터가 복잡해서 중간 점검이 필요할 때
- AI의 첫 분석이 방향이 틀렸을 때 빨리 교정하고 싶을 때
- 각 단계를 배우면서 진행하고 싶을 때

### `<!-- 💬 -->` 교육 주석 활용법

프롬프트 안에 세 가지 유형의 교육 주석이 있습니다:

```markdown
<!-- 💬 [전문가의 눈] 전략 분석가는 "사업 모델 → 경쟁우위 → 성장 벡터" 순서로 봅니다.
     왜? 사업 모델 없는 경쟁우위는 수익이 안 되기 때문. -->

<!-- 💬 [논쟁 지점] 리텐션을 D1으로 볼지 W1으로 볼지는
     앱 유형마다 전문가 의견이 다릅니다. -->

<!-- 💬 [자기 검증] 경쟁사 앱스토어 리뷰 10개를 직접 읽어
     AI 분석과 비교하세요. 일치도가 검증 기준입니다. -->
```

| 유형 | 의미 | 여러분에게 주는 것 |
|------|------|-----------------|
| `[전문가의 눈]` | 전문가는 왜 이 순서/프레임워크를 쓰는가 | 해석의 틀 |
| `[논쟁 지점]` | 전문가들도 의견이 갈리는 부분 | 불확실성의 인식 |
| `[자기 검증]` | 이 결과를 어떻게 테스트하는가 | 빈틈 발견 방법 |

이 주석은 AI에게 보내는 지시가 아닙니다. **여러분을 위한 설명**입니다. AI에 프롬프트를 붙여넣을 때 포함해도 되고 빼도 됩니다 (결과에 영향 없음).

### 자주 하는 실수

| 실수 | 올바른 방법 |
|------|------------|
| 전체를 한 번에 넣었는데 AI가 앞부분만 답함 | "아직 단계가 남았습니다. 이어서 N단계부터 계속하세요"라고 입력 |
| AI가 너무 일반적인 답을 줌 | 컨텍스트 섹션의 변수를 더 구체적으로 채우세요. 구체적 입력 = 구체적 출력 |
| 교육 주석을 AI 지시로 착각 | `<!-- -->` 안의 내용은 참고용입니다. AI에게 영향을 주지 않습니다 |
| 자기검증 단계를 건너뜀 | 마지막 "자기검증" 단계가 품질을 30-50% 높입니다. 꼭 포함하세요 |

### Layer 2에서 Layer 3로 넘어갈 신호

- "이 분석을 매주 반복해야 하는데..." → 자동화 필요 → Layer 3
- "여러 경쟁사를 한꺼번에 분석하고 싶은데..." → 배치 처리 필요 → Layer 3
- "이 결과를 다른 시스템에 연동하고 싶은데..." → API 연동 필요 → Layer 3

---

## 🔧 Layer 3: 자기 검증

### 이 레이어가 하는 일
읽었다고 아는 게 아닙니다. Layer 3는 Layer 1-2에서 세운 가설을 **코드로 실제 데이터에 부딪혀봅니다.** 스스로 테스트해봐야 빈틈이 보이고, 빈틈이 보여야 채울 수 있습니다.

### 대상
AI 엔지니어, 개발자, 자동화에 관심 있는 기술 리더

### Layer 3의 핵심: "이해를 코드로 검증하기"

Layer 1-2에서 세운 가설과 프레이밍을 **실제 데이터로 검증하는 파이프라인**으로 만듭니다.

```
Layer 2 (수동):                    Layer 3 (자동화):
리뷰 복사 → AI에 붙여넣기 →        리뷰 자동 수집 → API 호출 →
결과 정리 → 문서 작성               자동 분석 → 리포트 생성 → Slack 알림
(30분, 수동, 일회성)               (5분, 자동, 반복 가능)
```

### 시작 전 체크리스트

각 프롬프트의 Layer 3에 전제조건이 명시되어 있지만, 공통적으로 필요한 것:

| 항목 | 설명 | 확인 |
|------|------|------|
| Python 3.8+ | `python3 --version`으로 확인 | ☐ |
| pip 패키지 매니저 | `pip --version`으로 확인 | ☐ |
| Anthropic API 키 | [console.anthropic.com](https://console.anthropic.com)에서 발급 | ☐ |
| 환경변수 설정 | `export ANTHROPIC_API_KEY=your_key` | ☐ |
| (선택) Claude Code | [claude.ai/code](https://claude.ai/code) — CLI 자동화 시 필요 | ☐ |

### 코드 읽는 법

각 Layer 3 코드는 이런 구조를 따릅니다:

```python
# 1. 문서 상단: 이 코드가 뭘 하는지 설명
"""
경쟁사 감성 분석 자동화 파이프라인
- 리뷰 자동 수집 → 감성 분석 → 기회 도출 → 리포트
"""

# 2. 클래스/함수: 각 단계를 메서드로 분리
class Analyzer:
    def collect_data(self): ...    # Layer 2의 1-2단계
    def analyze(self): ...          # Layer 2의 3-5단계
    def generate_report(self): ...  # Layer 2의 6-8단계

# 3. 실행 예시: 바로 복사해서 실행 가능
if __name__ == "__main__":
    analyzer = Analyzer(["경쟁사1", "경쟁사2"])
    report = analyzer.generate_report()
```

### 자주 하는 실수

| 실수 | 올바른 방법 |
|------|------------|
| API 키 없이 실행 | 반드시 환경변수를 먼저 설정하세요 |
| 샘플 데이터로만 테스트 | 샘플 코드는 동작 확인용입니다. 실제 데이터로 교체하세요 |
| 코드를 수정 없이 그대로 사용 | Layer 3 코드는 "시작점"입니다. 여러분의 환경에 맞게 커스터마이징하세요 |
| 에러 발생 시 포기 | 에러 메시지를 Claude/ChatGPT에 붙여넣고 "이 에러 어떻게 해결하나요?"라고 물어보세요 |

## 💡 프롬프트 공통 팁

### 변수 채우기의 기술

프롬프트의 결과 품질은 **변수를 얼마나 구체적으로 채우느냐**에 달려 있습니다.

```
❌ 나쁜 예: {경쟁사명}의 분석
✅ 좋은 예: Notion (프로젝트 관리 + 문서 협업 SaaS, 시리즈 C, MAU 30M+)의 분석

❌ 나쁜 예: {AI 기능} 개선
✅ 좋은 예: "고객 문의 자동 분류 기능 (현재 정확도 85%, 목표 95%, 일 처리량 500건)"
```

변수에 넣으면 좋은 정보: 제품명, 산업, 규모, 현재 상태(숫자), 목표(숫자)

### AI별 차이

- **ChatGPT**: 창의적 답변에 강함. Layer 1 프롬프트에 가장 적합
- **Claude**: 긴 텍스트, 구조화된 분석에 강함. Layer 2 프롬프트에 적합
- **Gemini**: Google 데이터 연동에 강함. 실시간 정보가 필요한 분석에 적합

어떤 AI를 쓰든 결과는 나옵니다. 다만, **같은 프롬프트를 2개 이상의 AI에 넣어보면 교차 검증**이 됩니다.

### 결과를 개선하는 후속 질문 패턴

| 상황 | 후속 질문 |
|------|----------|
| 결과가 너무 일반적 | "각 항목에 구체적 수치와 근거를 추가하세요" |
| 형식을 바꾸고 싶음 | "위 내용을 마크다운 표 / 불릿포인트 / 슬라이드용으로 변환하세요" |
| 한국 맥락이 부족 | "한국 시장/사용자 관점에서 재해석하세요" |
| 빠진 관점이 있음 | "엔지니어 / 디자이너 / C-level 관점을 추가하세요" |
| 더 깊은 분석 필요 | "가장 중요한 상위 3개 항목만 깊이 분석하세요" |
| 실행 계획이 필요 | "이 분석 결과를 기반으로 2주간 실행 계획을 세우세요" |

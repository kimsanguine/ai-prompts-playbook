# 🧠 세 개의 질문 — AI 시대의 사고법

> 이 문서는 "프롬프트"가 아니라 **이 Playbook이 존재하는 이유**입니다.
> AI에게 무엇을 물어야 하는지 모르겠다면, 여기서 시작하세요.

---

## 같은 도구, 정반대의 결과

Harvard AI 튜터 실험(2024)에서 흥미로운 결과가 나왔습니다.

구조적 질문을 받은 학생은 49분 만에 60분 수업보다 **약 2배 높은 학습 성과**를 냈습니다. 반면, AI에게 답만 맡긴 학생들의 사고력은 **오히려 후퇴**했습니다.

같은 AI입니다. 같은 모델, 같은 시간, 같은 주제. 차이는 단 하나 — **뼈대를 세웠느냐, 답만 가져갔느냐.**

이 Playbook은 답을 모으는 도구가 아닙니다. **질문의 뼈대를 세우는 도구**입니다.

---

## 세 개의 질문

어떤 분야든, 다음 세 질문을 던지면 그 분야의 뼈대가 세워집니다.

### 질문 1: "전문가는 이걸 어떤 틀로 보나요?"

<!-- 💬 [전문가의 눈] 같은 데이터를 봐도 읽어내는 것이 다른 이유는
     지식의 양이 아니라 해석하는 틀의 차이입니다.
     이 틀을 먼저 세우면, 이후에 쌓이는 정보가 전부 제자리를 찾습니다. -->

```
❌ "경쟁사 분석해줘"
→ 위키피디아 수준의 나열

✅ "경쟁사 분석에서 전략 컨설턴트는 어떤 프레임워크를 쓰나요?
   그 프레임워크로 {경쟁사}를 분석하세요."
→ 전문가의 해석 틀이 먼저 세워지고, 분석이 그 위에 쌓임
```

**이 질문이 하는 일:** 정보의 파편이 아니라, 정보가 꽂힐 구조를 먼저 만듭니다.

### 질문 2: "전문가들도 답을 못 내린 부분은 어디인가요?"

<!-- 💬 [논쟁 지점] 교과서에 나오는 내용은 어디서든 찾을 수 있습니다.
     전문가들조차 답을 못 내린 지점을 아는 것이 진짜 이해의 시작입니다.
     확실한 것과 불확실한 것을 구분할 수 있어야 그 분야를 안다고 할 수 있습니다. -->

```
❌ "AI 윤리 체크리스트 만들어줘"
→ 교과서적 나열, 모든 항목이 같은 무게

✅ "AI 공정성 측정에서 demographic parity와 equalized odds 중
   어떤 기준을 써야 하는지 전문가들의 의견이 갈리는 지점은?"
→ 확실한 영역과 불확실한 영역의 경계가 보임
```

**이 질문이 하는 일:** 합의된 영역 너머를 봅니다. "모른다"는 것을 아는 것이 진짜 이해의 시작입니다.

### 질문 3: "내가 이해한 걸 어떻게 테스트하나요?"

<!-- 💬 [자기 검증] 읽었다고 아는 게 아닙니다.
     스스로 테스트해봐야 빈틈이 보이고, 빈틈이 보여야 채울 수 있습니다. -->

```
❌ AI 분석 결과를 그대로 보고서에 넣기
→ 답을 모으는 사람

✅ "이 분석에서 가장 확신이 낮은 부분 3개를 알려줘.
   그리고 그걸 10분 안에 직접 검증할 수 있는 방법도."
→ 공식을 가져가는 사람
```

**이 질문이 하는 일:** 자기 검증입니다. AI의 출력을 맹신하는 대신, 빈틈을 찾아 채웁니다.

## 세 질문과 세 레이어

이 Playbook의 3개 레이어는 난이도가 아닙니다. **세 질문의 구조**입니다.

```
┌─────────────────────────────────────────────────────────┐
│  ⚡ Layer 1: 전문가의 눈 빌리기                           │
│                                                         │
│  "이 분야의 전문가는 어떤 틀로 보는가?"                     │
│  → 복사-붙여넣기만으로 전문가의 프레이밍을 빌릴 수 있습니다   │
│  → 5분이면 정보가 꽂힐 구조가 세워집니다                    │
├─────────────────────────────────────────────────────────┤
│  🧠 Layer 2: 합의된 영역 너머 보기                        │
│                                                         │
│  "어디서 전문가들조차 답을 못 내리는가?"                     │
│  → CoT(Chain of Thought)로 확실한 것/불확실한 것을 구분     │
│  → 교육 주석이 "왜 이 단계인가"를 설명합니다                │
├─────────────────────────────────────────────────────────┤
│  🔧 Layer 3: 자기 검증                                   │
│                                                         │
│  "내가 이해한 것을 어떻게 증명하는가?"                      │
│  → 코드로 가설을 실제 데이터에 부딪혀봅니다                  │
│  → 이해한 것과 이해하지 못한 것의 경계가 명확해집니다         │
└─────────────────────────────────────────────────────────┘
```

### 레이어를 이렇게 읽으세요

| 기존 인식 | 새로운 인식 |
|----------|-----------|
| "Layer 1은 초보용" | "Layer 1은 전문가의 프레이밍을 빌리는 것" |
| "Layer 2는 PM용 심화" | "Layer 2는 확실한 것과 불확실한 것의 경계를 탐색하는 것" |
| "Layer 3는 개발자용" | "Layer 3는 내 이해를 코드로 검증하는 것" |
| "Layer 1 → 2 → 3로 올라감" | "세 질문을 모두 던져야 뼈대가 완성됨" |

**꼭 Layer 1부터 시작할 필요는 없습니다.** 하지만 세 질문을 모두 던져야 한 분야의 뼈대가 세워집니다.

## 교육 주석의 세 가지 표시

이 Playbook의 프롬프트에는 `<!-- 💬 -->` 교육 주석이 있습니다. 주석 안에 대괄호로 유형이 표시됩니다:

| 표시 | 의미 | 예시 |
|------|------|------|
| `[전문가의 눈]` | 전문가는 왜 이 순서/프레임워크를 쓰는가 | "전략 분석가는 사업 모델 → 경쟁우위 → 성장 벡터 순으로 봅니다" |
| `[논쟁 지점]` | 전문가들도 의견이 갈리는 부분 | "리텐션 기준을 D1으로 볼지 W1으로 볼지는 앱 유형에 따라 다릅니다" |
| `[자기 검증]` | 이 결과를 어떻게 테스트하는가 | "경쟁사 앱스토어 리뷰 10개를 직접 읽어 AI 분석과 비교하세요" |

**이 주석은 AI에게 보내는 지시가 아닙니다.** 여러분을 위한 설명입니다.
AI에 프롬프트를 붙여넣을 때 포함해도 되고 빼도 됩니다 (결과에 영향 없음).

하지만 — 이 주석을 읽으면 "왜 이 질문인가"를 이해하게 되고, 이해하면 어떤 분야에든 같은 구조를 적용할 수 있게 됩니다.

---

## 뼈대가 있으면 달라지는 것

```
뼈대 없이 AI를 쓰는 사람:
  "경쟁사 분석해줘" → 결과 복사 → 다음 질문 → 결과 복사 → ...
  → 파편이 쌓임. 새로운 정보가 들어오면 어디에 놓아야 할지 모름.

뼈대를 세우고 AI를 쓰는 사람:
  "전문가의 틀로 보면?" → 구조가 잡힘
  "의견이 갈리는 곳은?" → 확실한 것과 불확실한 것이 분리됨
  "검증하려면?" → 빈틈이 보이고, 채울 수 있음
  → 새로운 정보가 들어오면 어디에 꽂아야 하는지 보임.
```

이 세 질문은 특정 분야의 질문이 아닙니다. 의학에 던지면 의학의 뼈대가 나오고, 법학에 던지면 법학의 뼈대가 나옵니다. 마케팅이든 엔지니어링이든, 도메인이 바뀌어도 뼈대를 세우는 질문의 구조는 동일합니다.

## 이 Playbook과의 관계

이 Playbook의 33개 프롬프트는 PM 도메인에 세 질문을 적용한 것입니다.

```
세 질문 (범용)              이 Playbook (PM 도메인)
───────────────           ────────────────────
전문가의 눈 빌리기    →    Layer 1 프롬프트 (전문가의 프레이밍)
합의 너머 보기       →    Layer 2 CoT + [논쟁 지점] 주석
자기 검증           →    Layer 3 코드 + [자기 검증] 주석
```

**복사-붙여넣기만 해도 결과는 나옵니다.** 하지만 "왜 이 질문인가"를 이해하면:

1. 이 Playbook에 없는 분야도 같은 구조로 질문할 수 있습니다
2. AI의 출력을 맹신하는 대신, 빈틈을 찾아 채울 수 있습니다
3. 시간이 갈수록 질문하는 능력이 쌓여, AI가 더 강력한 증폭기가 됩니다

> 도메인 지식은 이제 AI로 압축됩니다. 그런데 **어떤 질문을 던질지 아는 능력**은 압축되지 않습니다. 같은 AI를 쓰면서 누군가는 답을 모으고, 누군가는 공식을 가져옵니다. 이 차이는 시간이 갈수록 벌어집니다.

# 🚪 제품 종료 메시지 (Product End-of-Life Communication)

> 예상 소요: 10분 (Quick) / 25분 (Deep) / 40분+ (Build)
> 워크플로우: 독립 → 마케팅/고객 지원팀과 함께 실행

---

## 🎯 이 프롬프트가 해결하는 문제

제품 종료(EOL, End-of-Life) 공지는 **가장 섬세한 커뮤니케이션**입니다. 잘못하면 고객 신뢰를 잃고, 법적 문제까지 발생할 수 있습니다.
**한국 시장 특화된 EOL 커뮤니케이션 가이드**로 고객을 존중하면서도 기업의 의도를 명확히 전달하세요.
이는 단순한 "안녕히"가 아니라, 고객을 새로운 솔루션으로 **부드럽게 전환**시키는 전략입니다.

---

## 📥 이전 프롬프트에서 받는 입력

> 단독 사용 가능. 제품/기능 종료 결정 후 고객 공지를 작성할 때 사용합니다.

**종료 대상 기능/제품, 종료 일정, 대체 방안, 영향 받는 사용자 수**를 직접 입력하세요.

## ⚡ Layer 1: Quick Start

> **이 레이어가 하는 일: 전문가의 눈 빌리기**
> 전문가가 이 분야를 볼 때 쓰는 프레이밍을 빌려줍니다. AI를 처음 쓰는 분, 비개발자, 학생 — 5분이면 제품/기능 종료 공지를 작성할 수 있습니다.

### 📌 사용법

1. **ChatGPT, Claude, Gemini 중 아무 AI 챗봇**을 열고 새 대화를 시작하세요
2. 아래 프롬프트를 **통째로 복사**해서 AI에 붙여넣으세요
3. `{중괄호}` 안의 내용만 여러분의 상황에 맞게 바꾸세요
4. Enter를 누르면 끝!

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


```
다음을 입력하세요:
- 종료할 제품명
- 종료 이유 (기술 부채, 새 제품으로 대체, 사업 축소 등)
- 최종 지원 날짜 (예: 2024년 12월 31일)
- 고객 수, 중요도 (VIP vs 일반)

자동으로 생성됩니다:
1. 공식 공지문
2. 고객 이메일 (개인화)
3. FAQ
4. 마이그레이션 가이드
5. 지원팀 교육 자료

한국 법규 준수 + 감정 배려 포함
```

💡 **Quick Tips:**
- 최소 **6개월 전** 공지하세요 (한국 시장 기준)
- 고객과의 **계약 의무**를 먼저 확인하세요
- 무료 마이그레이션 지원이 신뢰를 만듭니다


### ⚠️ 이런 경우는요?

| 상황 | 대처법 |
|------|--------|
| 긴급 EOL (보안 이슈) | 안전 우선으로 즉시 종료 공지. 대안과 지원을 최대한 빨리 제공 |
| 유료 고객이 영향 받음 | 환불/크레딧 정책을 먼저 확정 → 메시지에 포함 |
| 대안 제품이 없음 | "왜 종료하는지" 투명하게 설명 + 데이터 백업 방법 안내 |
| 내부 반발이 있음 | stakeholder-alignment.md로 먼저 내부 합의 → 그 후 외부 공지 |

## 🧠 Layer 2: Deep Dive

> **이 레이어는 누구를 위한 건가요?**
> PM, 기획자, 마케터 — 제품/기능 종료를 고객에게 부드럽게 전달하고 싶은 분입니다.
고객 이탈을 최소화하면서 EOL을 진행하고 싶다면 이걸 써보세요.

### 📌 진행 방식

이 프롬프트는 **4단계 CoT(Chain of Thought)** 구조입니다. 두 가지 방식으로 쓸 수 있습니다:

**방식 A: 한 번에 넣기 (추천)**
- 아래 전체를 통째로 복사해서 AI에 붙여넣으세요
- AI가 모든 단계를 순서대로 실행합니다
- 결과가 한 번에 나옵니다 (길지만 완결적)

**방식 B: 단계별로 넣기 (더 깊은 분석)**
- 1-2단계를 먼저 넣고 결과를 확인하세요
- 결과가 만족스러우면 다음 단계를 이어서 넣으세요
- 중간에 수정이 필요하면 해당 단계를 수정 요청할 수 있습니다
- 시간이 더 걸리지만, 각 단계를 직접 검증할 수 있습니다

**`<!-- 💬 -->` 교육 주석이란?**
- 프롬프트 안에 `<!-- 💬 ... -->` 형태의 주석이 있습니다
- 이건 **여러분을 위한 설명**이지, AI에게 보내는 지시가 아닙니다
- AI에 붙여넣을 때 포함해도 되고 빼도 됩니다 (결과에 영향 없음)
- "왜 이렇게 설계했는가?"를 이해하면 더 좋은 프롬프트를 만들 수 있습니다

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.

### # Reasoning Strategy: EOL 커뮤니케이션 5단계

#### 1단계: 종료 공지 타이밍과 기간

**한국 시장 기준:**

```
이상적 타임라인:

Day 0: 공식 공지
       ├─ 블로그 포스트
       ├─ 이메일 (모든 고객)
       ├─ 지원팀 교육
       └─ FAQ 공개

Month 1-2: 조용한 준비 기간
       ├─ 신제품 장점 설명
       ├─ 마이그레이션 도구 준비
       └─ VIP 고객 1:1 상담

Month 3-5: 적극적 마이그레이션 지원
       ├─ 무료 마이그레이션 서비스
       ├─ 교육 웨비나
       └─ 데이터 이전 도구

Month 6: 최종 공지
       └─ "X개월 남음" 리마인더

Final: 종료 (데이터 백업 옵션 제공)

법적 최소 기간:
- 개인정보 포함: 6개월 (개인정보보호법)
- 엔터프라이즈: 12개월 (계약 관례)
- SaaS: 6개월 (업계 표준)
```

#### 2단계: 고객 세그먼트별 메시지 차별화

다른 고객에게는 다른 메시지가 필요합니다.

```
고객 세그먼트:

1️⃣ VIP 고객 (연 계약금 1억+, 중요도 높음)
   메시지: "당신은 우리에게 중요합니다"
   - 1:1 전담 PM 배정
   - 맞춤형 마이그레이션 계획
   - 우선 지원, 무료 교육
   - 예: "귀사의 특별한 요구사항을 위해 맞춤 솔루션을 준비했습니다"

2️⃣ 일반 고객 (월 계약, 주요 고객)
   메시지: "당신을 위한 더 나은 솔루션입니다"
   - 그룹 웨비나
   - 셀프 서비스 마이그레이션 도구
   - 메일 지원
   - 예: "[신제품]의 3배 빠른 속도, 50% 저렴한 비용"

3️⃣ 프리 사용자
   메시지: "감사합니다, 도움을 드립니다"
   - 자동 이메일 + FAQ
   - 커뮤니티 포럼 지원
   - 예: "프리 플랜이셨는데, [신제품]에서도 무료로 쓸 수 있습니다"

4️⃣ 미사용 고객 (1년 이상 로그인 안 함)
   메시지: "간단한 안내"
   - 자동 이메일
   - 의도적 답장 없어도 괜찮음
```

#### 3단계: 공식 공지문 작성

**법적 요구사항 + 감정 배려**

```
필수 포함 사항:

✓ 종료 제품명과 날짜
✓ 종료 이유 (간단히)
✓ 고객에게 미치는 영향 (구체적)
✓ 대체 솔루션 제안
✓ 마이그레이션 지원 내용 (무료 여부, 기간)
✓ 데이터 보관/삭제 옵션
✓ 연락처와 지원 방법
✓ 다음 단계 (액션 아이템)

예시 구조:

---
**[제품명] 종료 안내**

안녕하세요, [고객명]님.

저희는 지난 [XX년] 동안 [제품명]을 제공해드렸습니다.
[고객명]님의 사랑으로 [사용자 수], [거래량] 등의 성과를 이뤘습니다.

그러나 시장의 변화와 기술 발전에 따라,
저희는 [신제품]으로 한 단계 나아가기로 결정했습니다.

**[신제품]의 장점:**
- 3배 빠른 처리 속도
- 50% 저렴한 비용
- [차별점 3개]

**[제품명] 종료 일정:**
- 2024년 12월 31일까지 지원
- 로그인은 2025년 1월 15일까지 가능
- 데이터는 2025년 2월까지 다운로드 가능

**저희가 도와드리는 것:**
- 무료 마이그레이션 (개인화, 데이터 이전 포함)
- 1:1 전담 지원 (귀사 규모에 따라)
- [신제품] 무료 1개월 추가 이용

**귀하의 데이터:**
- 모든 데이터는 안전하게 보관됩니다
- 언제든 CSV, JSON 등으로 다운로드 가능
- 12개월 후 자동 삭제됩니다 (요청 시 연장)

**다음 단계:**
1. 아래 링크에서 마이그레이션 계획 확인
2. 2024년 9월 30일까지 신청
3. 저희 팀이 연락드립니다

[마이그레이션 신청 링크]
[FAQ 링크]
[지원팀 연락처]

감사합니다, [회사명]

---
```

#### 4단계: 마이그레이션 지원 계획

**고객이 성공하도록 구체적 지원**

```
지원 내용:

1. 데이터 마이그레이션
   ├─ 자동 데이터 이전 도구
   ├─ 매핑 검증 (손실 없음)
   └─ 무료 사전 테스트

2. 시스템 통합
   ├─ API 문서 (신제품)
   ├─ 기존 시스템과의 연동 가이드
   └─ 통합 테스트 지원

3. 교육 & 온보딩
   ├─ 그룹 웨비나 (녹화 제공)
   ├─ 1:1 교육 (VIP 고객)
   ├─ 사용자 가이드 (한글)
   └─ 기술 FAQ

4. 시간 제한 인센티브
   ├─ 2024년 9월까지 신청 시: 무료 마이그레이션
   ├─ 2024년 10월~11월: 반값 마이그레이션
   ├─ 2024년 12월: 유료 마이그레이션
   └─ 2025년 1월: 자가 마이그레이션만 가능

5. 비용 지원
   ├─ 소규모 고객: 전액 무료
   ├─ 중규모 고객: 50% 비용 지원
   ├─ VIP 고객: 100% 무료 + 추가 서비스
```

#### 5단계: 위기 관리 시나리오

**나쁜 반응에 대비**

```
예상되는 고객 반응과 대응:

😡 "갑자기 종료? 신뢰할 수 없다"
   → 대응: 공지 6개월 전부터 시작했음 강조
           합의 계약 확인, 법적 의무 준수 증명

😔 "비용이 너무 많이 든다"
   → 대응: 무료 마이그레이션 옵션 강조
           [신제품]의 비용 절감 장점 설명

🤔 "정말 필요한가?"
   → 대응: 기술 부채, 보안 위험 등 구체적 이유
           [신제품]의 새로운 기능 소개

❌ "마이그레이션 못 한다"
   → 대응: 1:1 전담 지원 제안
           분할 마이그레이션 옵션
           延長(연장) 시간 협의

대응 원칙:
✓ 고객의 걱정을 이해하는 태도
✓ 구체적 해결책 제시
✓ 일관된 메시지 전달
✓ 투명성 유지
```

### # 한국 시장 특화

```
한국 특화 고려사항:

1. 법적 요구사항
   - 개인정보보호법: 6개월 사전 공지
   - 정보통신망법: 서비스 중단 통지
   - 계약법: 계약서의 해지 조건 검토

2. 문화적 감수성
   - "고마워요" 강조 (서구와 다름)
   - 고객과의 장기 관계 존중
   - 신뢰 회복이 중요

3. 지원팀 교육
   - "왜?" 질문에 답할 수 있도록
   - 감정 공감 교육
   - 마이그레이션 상담 능력

4. 다중 채널
   - 이메일 (공식)
   - 인앱 팝업 (긴급)
   - 전화 (VIP)
   - 블로그 (투명성)
   - SNS (커뮤니티 지원)
```

### 📌 시작 전에

**이 코드가 하는 일:**
Layer 1-2에서 수동으로 했던 과정을 Python 스크립트로 자동화합니다.

**왜 자동화하나?**
EOL은 여러 단계의 공지가 필요합니다 (사전 공지 → 마이그레이션 안내 → 최종 종료). 자동화하면 일정에 맞춰 정확한 메시지가 발송됩니다.

**전제조건:**
- Python 3.8+ 설치
- 필요한 패키지 설치 (코드 상단의 import 참고)
- Anthropic API 키 발급 → 환경변수 설정: `export ANTHROPIC_API_KEY=your_key`

> 📖 더 자세한 설정 방법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


### EOL 커뮤니케이션 자동화 도구

```python
#!/usr/bin/env python3
"""
제품 EOL 커뮤니케이션 자동 생성 및 관리 도구
고객 세그먼트별 맞춤형 메시지 생성
"""

import anthropic
import json
from datetime import datetime, timedelta
from typing import Optional
from enum import Enum


class CustomerSegment(Enum):
    """고객 세그먼트"""
    VIP = "vip"  # 연 계약금 1억+
    STANDARD = "standard"  # 주요 고객
    FREE = "free"  # 프리 사용자
    INACTIVE = "inactive"  # 비활성 사용자


@dataclass
class EOLConfig:
    """EOL 설정"""
    product_name: str
    end_date: str  # "2024-12-31"
    replacement_product: str
    reason: str  # "기술 부채", "새 제품 대체" 등
    migration_support_days: int  # 마이그레이션 지원 기간


def generate_eol_messages(config: EOLConfig) -> dict:
    """
    EOL 메시지 자동 생성

    Args:
        config: EOL 설정

    Returns:
        고객 세그먼트별 메시지 및 구현 계획
    """

    client = anthropic.Anthropic()

    prompt = f"""당신은 제품 마케팅 및 고객 지원 전문가입니다.
한국 시장을 이해하고, 고객 신뢰를 지키는 EOL 커뮤니케이션을 만드세요.

**EOL 정보:**
- 종료 제품: {config.product_name}
- 종료 날짜: {config.end_date}
- 대체 제품: {config.replacement_product}
- 종료 이유: {config.reason}
- 마이그레이션 지원 기간: {config.migration_support_days}일

**생성할 콘텐츠:**

1. **공식 공지문** (블로그/이메일)
   - 고객 감정 배려
   - 구체적 일정
   - 법적 의무 명시
   - 지원 내용

2. **고객 세그먼트별 이메일**

   a) VIP 고객 (연 1억+)
      - 전담 PM 배정 제안
      - 맞춤형 마이그레이션
      - 1:1 통화 제안

   b) 일반 고객 (주요)
      - 그룹 웨비나 안내
      - 무료 마이그레이션 강조
      - 비용 절감 메시지

   c) 프리 사용자
      - 감사 메시지
      - 신제품 무료 체험 링크
      - 간단한 다음 단계

   d) 비활성 사용자
      - 짧고 친절한 안내
      - 데이터 다운로드 옵션

3. **FAQ** (최소 10개)
   - "왜 제품을 종료하나요?"
   - "데이터는 어떻게 되나요?"
   - "비용은 들지 않나요?"
   - "다른 솔루션으로는?"
   등등

4. **마이그레이션 가이드**
   - 3단계 프로세스
   - 각 단계별 걸리는 시간
   - 지원 받는 방법

5. **지원팀 교육 자료**
   - 고객 질문별 대응
   - 감정 공감 표현
   - 에스컬레이션 기준

**JSON 응답 형식:**

{{
  "official_announcement": {{
    "title": "제목",
    "content": "본문 (마크다운 형식)",
    "tone": "empathetic"
  }},
  "email_templates": {{
    "vip": {{
      "subject": "제목",
      "body": "본문",
      "cta": "행동 유도 버튼"
    }},
    "standard": {...},
    "free": {...},
    "inactive": {...}
  }},
  "faqs": [
    {{"q": "질문", "a": "답변", "segment": "all"}}
  ],
  "migration_guide": {{
    "step1": "단계 설명",
    "timeline": "소요 시간"
  }},
  "support_team_brief": {{
    "key_messages": ["메시지1", "메시지2"],
    "common_objections": [
      {{"objection": "고객 우려", "response": "대응"}}
    ]
  }},
  "timeline": {{
    "day0": "공지",
    "month1": "활동",
    "final": "종료"
  }}
}}

한글로 작성하세요.
"""

    message = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=4000,
        messages=[{"role": "user", "content": prompt}]
    )

    try:
        import re
        json_match = re.search(r'\{.*\}', message.content[0].text, re.DOTALL)
        if json_match:
            result = json.loads(json_match.group())
        else:
            result = {"raw": message.content[0].text}
    except json.JSONDecodeError:
        result = {"raw": message.content[0].text}

    return {
        "product": config.product_name,
        "end_date": config.end_date,
        "generated_content": result,
        "tokens_used": message.usage.output_tokens
    }


def generate_timeline(config: EOLConfig) -> list:
    """EOL 타임라인 생성"""

    end_date = datetime.strptime(config.end_date, "%Y-%m-%d")
    today = datetime.now()
    days_until_eol = (end_date - today).days

    timeline = [
        {
            "date": today.strftime("%Y-%m-%d"),
            "event": "공식 공지",
            "action": "모든 고객에게 이메일 발송"
        },
        {
            "date": (today + timedelta(days=30)).strftime("%Y-%m-%d"),
            "event": "마이그레이션 신청 시작",
            "action": "무료 마이그레이션 신청 링크 활성화"
        },
        {
            "date": (end_date - timedelta(days=90)).strftime("%Y-%m-%d"),
            "event": "마이그레이션 신청 마감",
            "action": "최종 공지, 유료 옵션 전환"
        },
        {
            "date": (end_date - timedelta(days=14)).strftime("%Y-%m-%d"),
            "event": "최종 리마인더",
            "action": "아직 미사용 고객 집중 대상"
        },
        {
            "date": end_date.strftime("%Y-%m-%d"),
            "event": "종료",
            "action": "서비스 중지, 데이터 보관 시작"
        }
    ]

    return timeline


if __name__ == "__main__":
    # 예제
    config = EOLConfig(
        product_name="데이터 분석 Classic",
        end_date="2024-12-31",
        replacement_product="데이터 분석 Pro",
        reason="새로운 AI 기능 포함 및 3배 빠른 성능으로 발전",
        migration_support_days=180
    )

    result = generate_eol_messages(config)
    print(json.dumps(result, ensure_ascii=False, indent=2))

    # 타임라인 생성
    timeline = generate_timeline(config)
    print("\n=== EOL TIMELINE ===")
    for item in timeline:
        print(f"{item['date']}: {item['event']} → {item['action']}")
```

## 🔍 이 결과를 받은 후

> 결과를 그대로 쓰기 전에, 전문가라면 던질 세 가지 질문입니다.

1. **[전문가의 눈]** 공지문에서 '감사'와 '대안'이 구체적인지 확인하세요 — 막연한 감사는 역효과
2. **[논쟁 지점]** 종료 공지를 '한 번에 보낼지' vs '단계적으로 보낼지'는 사용자 규모에 따라 다릅니다
3. **[자기 검증]** 종료 공지문을 현재 해당 기능의 헤비 유저 1명에게 보여주고 반응을 확인하세요

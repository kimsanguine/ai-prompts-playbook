# 📍 포지셔닝 전략 (Positioning Strategy)

> 예상 소요: 5분 (Quick) / 20분 (Deep) / 30분+ (Build)
> 워크플로우: → `decision-canvas.md` → `feature-prioritization.md`

---

## 🎯 이 프롬프트가 해결하는 문제

AI 제품을 시장에 출시할 때, 기존 솔루션 대비 차별점을 명확히 정의하지 못하면 시장에서 `"또 다른 AI 도구"` 취급을 받게 됩니다. 포지셔닝 전략은 **제품의 고유한 가치 명제(Value Proposition)를 시장 내 정확히 배치**하는 프롬프트입니다.

특히 AI 제품은 기술 혁신 자체가 포지셔닝이 아니라, **누가, 어떤 문제를 해결하는가**에 집중해야 합니다.

---

## 📥 이전 프롬프트에서 받는 입력

> `competitor-sentiment.md`에서 경쟁사 약점을 파악한 후, 차별화된 포지셔닝을 수립할 때 연결됩니다.

이전 프롬프트 결과에서 **경쟁사 약점, 사용자 니즈, 시장 기회**를 가져옵니다.

---

## ⚡ Layer 1: Quick Start

> **이 레이어가 하는 일: 전문가의 눈 빌리기**
> 전문가가 이 분야를 볼 때 쓰는 프레이밍을 빌려줍니다. AI를 처음 쓰는 분, 비개발자, 학생 — 5분이면 제품의 핵심 포지셔닝을 정리할 수 있습니다.

### 📌 사용법

1. **ChatGPT, Claude, Gemini 중 아무 AI 챗봇**을 열고 새 대화를 시작하세요
2. 아래 프롬프트를 **통째로 복사**해서 AI에 붙여넣으세요
3. `{중괄호}` 안의 내용만 여러분의 상황에 맞게 바꾸세요
4. Enter를 누르면 끝!

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


```
당신은 AI 제품 포지셔닝 전략가입니다.

{PRODUCT_NAME}의 포지셔닝 명제를 다음 구조로 작성하세요:

1. 타겟 고객: [구체적 직무/산업]
2. 고객의 핵심 문제: [2-3개 통증점]
3. 우리의 솔루션: [AI 기술 없이 비즈니스 임팩트로 설명]
4. 경쟁사 대비 차별점: [3개 구체적 차이점]
5. 포지셔닝 문장: [25단어 이내, 마케팅용]

{CONTEXT}
```

💡 **Quick Tips:**
- "AI"라는 단어를 포지셔닝에 포함하지 마세요 (기술이 아니라 결과에 집중)
- 경쟁사 분석 시 직접 경쟁사(Competitor)가 아닌 **대체재(Alternative)**도 포함하세요
- 포지셔닝은 1-2개 문제에 깊이 있게, 많은 문제를 해결한다고 주장하지 마세요


### ⚠️ 이런 경우는요?

| 상황 | 대처법 |
|------|--------|
| 경쟁사와 차별점이 안 보임 | 고객 리뷰에서 "왜 우리를 선택했나?"를 먼저 분석하세요 |
| B2B vs B2C 포지셔닝 차이 | B2B는 ROI/효율성, B2C는 감성/편의성에 초점. 분리해서 작성 |
| 여러 세그먼트에 다른 포지셔닝 필요 | 핵심 메시지는 통일하되, 세그먼트별 강조점만 다르게 |
| 기술 우위만 있고 비즈니스 가치가 약함 | "기술 → 고객 혜택 → 비즈니스 임팩트" 순으로 번역하세요 |

## 🧠 Layer 2: Deep Dive

> **이 레이어는 누구를 위한 건가요?**
> PM, 기획자, 마케터 — 포지셔닝 전략을 체계적으로 수립하고 싶은 분입니다.
"우리 제품이 왜 특별한가?"를 명확한 프레임워크로 정리하고 싶다면 이걸 써보세요.

### 📌 진행 방식

이 프롬프트는 **5단계 CoT(Chain of Thought)** 구조입니다. 두 가지 방식으로 쓸 수 있습니다:

**방식 A: 한 번에 넣기 (추천)**
- 아래 전체를 통째로 복사해서 AI에 붙여넣으세요
- AI가 모든 단계를 순서대로 실행합니다
- 결과가 한 번에 나옵니다 (길지만 완결적)

**방식 B: 단계별로 넣기 (더 깊은 분석)**
- 1-2단계를 먼저 넣고 결과를 확인하세요
- 결과가 만족스러우면 다음 단계를 이어서 넣으세요
- 중간에 수정이 필요하면 해당 단계를 수정 요청할 수 있습니다
- 시간이 더 걸리지만, 각 단계를 직접 검증할 수 있습니다

**`<!-- 💬 -->` 교육 주석이란?**
- 프롬프트 안에 `<!-- 💬 ... -->` 형태의 주석이 있습니다
- 이건 **여러분을 위한 설명**이지, AI에게 보내는 지시가 아닙니다
- AI에 붙여넣을 때 포함해도 되고 빼도 됩니다 (결과에 영향 없음)
- "왜 이렇게 설계했는가?"를 이해하면 더 좋은 프롬프트를 만들 수 있습니다

> 📖 더 자세한 사용법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.

```
## Step 3: 가치 명제(Value Proposition) 개발
```
다음 구조로 가치 명제를 개발하세요:

**포지셔닝 문장 (25단어 이내)**
[타겟 고객]을 위해 [경쟁사]와 달리 [차별점]으로 [가치]를 제공합니다.

예시 (AI 고객 지원 솔루션):
"기술 팀이 아닌 CS 팀을 위해 코딩 없이 AI 챗봇을 구축할 수 있으므로
30일 내 ROI를 달성할 수 있습니다."

**확장 설명 (3-4 문장)**
왜 이 포지셔닝인지 설명하세요:
1. 시장 기회 (TAM, 성장률)
2. 고객 니즈 (해결할 문제)
3. 우리의 능력 (기술, 팀, 리소스)

고객 관점에서 "정확도 99.5% → 수동 검수 시간 50% 감소 → 월 50만 원 절감"처럼 변환하세요.

```

### 🇰🇷 한국 시장 포지셔닝 특화

```
한국 SaaS/AI 시장에서의 포지셔닝 전략:

1. **한국 시장 포지셔닝 지형 (글로벌 vs 로컬)**:
   - 글로벌 vs 로컬 대결 구도: Intercom(글로벌) vs Zendesk(글로벌) vs 카톡톡(로컬)
   - 포지셔닝 기회: "글로벌 기술 + 한국 고객 특화" (예: 전자계약, 한글 NLP)
   - 위험: "한국만 타겟"은 Scale 한계 → 초기부터 글로벌 확장성 포함
   - 조정: V1(한국 특화) → V2(동남아) → V3(글로벌) 로드맵

2. **한국 마케팅 채널 특성**:
   - 네이버 검색 최강: 네이버 검색(50% 점유율) > Google(30%)
   - 카카오 광고: 카톡 버튼/광고가 가장 효율적 (CTR 높음)
   - 디스콰이엇(커뮤니티): 개발자/기업가 타겟팅 효율
   - 지피터스(금융): 금융 고객 타겟팅 전용 채널
   - 포지셔닝 메시지: "SEO 친화적 + 카톡 공유 가능" 구조 필수

3. **한국 기업 고객 구매 의사결정 특성**:
   - 기업 품의/결재 프로세스: PM → 팀장 → 부서장 → CFO (결정 4-6주)
   - 의사결정 기준: ROI(비용 절감) > 기술(혁신)
   - 신뢰 구축: 유명한 기업 고객(레퍼런스) + 공인 자격증(ISO, CSAP)
   - 포지셔닝 강점: "비용 절감 구체화" (월 500만 원 절감 예시)

4. **한국 시장 진입 전략 패턴**:
   - 한국어 지원이 첫 번째 허들: 영문만으로는 Reach 50% 이하
   - 결제 시스템: 한국 결제사(INICIS, 핑크이즈) 필수 통합 (비용 3-5만 원)
   - 고객 지원: 한국어 채팅 지원 필수 (영어만 지원 시 이탈 높음)
   - 초기 고객: 한국 스타트업(네이버/라인/당근 출신) → 대기업 확장

5. **한국 업종별 포지셔닝 차이**:
   - 금융: 보안/컴플라이언스 중심 (CSAP, 개보법 준수 강조)
   - 커머스: 속도/유연성 중심 (빠른 배포, A/B 테스트 용이)
   - 제조: 가격/신뢰성 중심 (비용 절감 증명, 장기 지원 보증)
   - 포지셔닝 조정: 산업별로 메시지 차별화 (공통 기술, 다른 가치 제안)
```

## Step 4: 포지셔닝 검증 (Product Trio 협의)
```
Product Trio와 함께 다음을 검증하세요:

**PM:**
- 이 포지셔닝으로 TAM 구매 가능한가?
- 경쟁사는 이 위치를 어떻게 방어할 것인가?

**Designer:**
- 이 포지셔닝을 브랜드/UI/메시지로 어떻게 구현할 것인가?
- 타겟 고객이 5초 만에 이해할 수 있는가?

**Engineer:**
- 이 포지셔닝을 기술로 3-6개월 내에 구현 가능한가?
- 장기적 기술 호탱성(sustainability)이 있는가?

<!-- 💬 Product Trio 동의가 없으면 실패합니다
포지셔닝은 마케팅 전략이 아니라 **제품 전략**입니다.
Designer가 UI로 구현 불가능하거나, Engineer가 기술적으로 불가능하면
시장의 아무리 큰 기회도 잡을 수 없습니다.
-->
```

# 예시: AI 고객 지원 솔루션의 포지셔닝

## V1 (Early Market): 기술 주도
- 타겟: AI/ML 엔지니어가 있는 대규모 회사
- 포지셔닝: "맞춤형 AI 모델을 한 주 내에 배포할 수 있습니다"
- 차별점: 모델 학습 속도, 정확도

## V2 (Growth Market): 비즈니스 주도
- 타겟: CS 팀장/매니저
- 포지셔닝: "코딩 없이 고객 이슈 70% 자동 해결하세요"
- 차별점: 사용 편의성, 빠른 ROI

## V3 (Mature Market): 규제/신뢰 주도
- 타겟: Fortune 500 금융사 CRO
- 포지셔닝: "감사 추적, 설명 가능성, GDPR 준수가 기본으로 제공됩니다"
- 차별점: 컴플라이언스, 보안, SLA

💡 **Deep Tips:**
- AI 제품은 **신뢰 빌드가 포지셔닝**입니다. 초기에는 "5초 더 빠르다"가 아니라 "이전 시스템과 100% 호환이 된다"로 포지셔닝하세요
- 한국 SaaS는 해외 경쟁사(Intercom, Zendesk)와 직접 비교하기보다 **국내 고객의 특수한 요구사항**(한글 자연어 처리, 전자계약법 준수)에 집중하세요
- 포지셔닝은 분기마다 리뷰하세요. 3-6개월마다 경쟁 지형이 변합니다
```

**왜 자동화하나?**
포지셔닝은 시장 변화에 따라 업데이트해야 합니다. 경쟁사 분석 데이터를 입력하면 자동으로 포지셔닝 맵을 생성합니다.

**전제조건:**
- Python 3.8+ 설치
- 필요한 패키지 설치 (코드 상단의 import 참고)
- Anthropic API 키 발급 → 환경변수 설정: `export ANTHROPIC_API_KEY=your_key`

> 📖 더 자세한 설정 방법은 [프롬프트 사용 가이드](../meta/how-to-use.md)를 참고하세요.


### Python 스크립트: 포지셔닝 분석 자동화

```python
#!/usr/bin/env python3
"""
AI 제품 포지셔닝 분석 자동화
- 경쟁사 지형도 생성
- 포지셔닝 강점/약점 분석
- Product Trio 피드백 수집
"""

import json
from anthropic import Anthropic

client = Anthropic()

def analyze_positioning(product_name, context):
    """포지셔닝 분석을 위한 대화형 프롬프트"""

    conversation = []
    system_prompt = """당신은 AI 제품 포지셔닝 전략가입니다.
다음 단계로 진행하세요:
1. 타겟 고객 세분화
2. 경쟁 지형도 작성
3. 가치 명제 개발
4. Product Trio 관점 검증

각 단계마다 JSON 형식으로 응답하세요."""

    # Step 1: 타겟 고객 분석
    conversation.append({
        "role": "user",
        "content": f"""제품명: {product_name}
컨텍스트: {context}

Step 1: 타겟 고객을 다음 6가지로 분류해주세요:
1. 산업
2. 직무
3. 회사 규모
4. 성숙도
5. 가치 동인
6. 구매 권한자

JSON 형식으로 응답하세요."""
    })

    response1 = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=1500,
        system=system_prompt,
        messages=conversation
    )

    conversation.append({
        "role": "assistant",
        "content": response1.content[0].text
    })

    print("📊 타겟 고객 분석:")
    print(response1.content[0].text)
    print("\n" + "="*60 + "\n")

    # Step 2: 경쟁 지형도
    conversation.append({
        "role": "user",
        "content": f"""Step 2: 경쟁 지형도를 2x2 매트릭스로 작성해주세요.
축:
- X축: 기술 성숙도 (Low → High)
- Y축: 사용자 경험 (Low → High)

직접 경쟁사 3개, 대체재 3개, {product_name}의 목표 포지션을 포함하세요.
JSON 형식으로 응답하세요."""
    })

    response2 = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=1500,
        system=system_prompt,
        messages=conversation
    )

    conversation.append({
        "role": "assistant",
        "content": response2.content[0].text
    })

    print("🎯 경쟁 지형도:")
    print(response2.content[0].text)
    print("\n" + "="*60 + "\n")

    # Step 3: 가치 명제
    conversation.append({
        "role": "user",
        "content": f"""Step 3: 가치 명제를 다음으로 개발해주세요:
- 포지셔닝 문장 (25단어 이내)
- 확장 설명 (3-4 문장): 시장 기회, 고객 니즈, 우리의 능력

JSON 형식으로 응답하세요."""
    })

    response3 = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=1500,
        system=system_prompt,
        messages=conversation
    )

    conversation.append({
        "role": "assistant",
        "content": response3.content[0].text
    })

    print("💡 가치 명제:")
    print(response3.content[0].text)
    print("\n" + "="*60 + "\n")

    # Step 4: Product Trio 검증
    conversation.append({
        "role": "user",
        "content": """Step 4: Product Trio 관점에서 검증해주세요.
각 역할(PM, Designer, Engineer)의 위험 요소와 기회를 분석하세요.

JSON 형식으로 응답하세요."""
    })

    response4 = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=2000,
        system=system_prompt,
        messages=conversation
    )

    print("🔍 Product Trio 검증:")
    print(response4.content[0].text)

# 실행
if __name__ == "__main__":
    product_name = "CustomerAI"  # 제품명 입력
    context = "고객 지원팀이 AI를 활용해 자동 응답 시스템을 구축하는 플랫폼"

    analyze_positioning(product_name, context)
```

### 제품: CustomerAI (고객 지원 AI 플랫폼)

**타겟 고객:**
- 산업: SaaS, 전자상거래
- 직무: CS 팀장, 고객 지원 담당자
- 회사 규모: Mid-Market (직원 50-500명)
- 성숙도: Early Majority
- 가치 동인: 응답 시간 단축, 비용 절감
- 구매 권한자: CS Director, VP

**경쟁 지형도:**

| | 기술 성숙도 Low | 기술 성숙도 High |
|---|---|---|
| **UX High** | Intercom AI (대체재) | **CustomerAI** ✓ |
| **UX Low** | Rasa (경쟁사) | Zendesk Einstein (경쟁사) |

**포지셔닝 문장:**
"CS 팀을 위해 노코드로 고객 이슈 70% 자동 해결하고, 30일 내 ROI를 달성합니다."

**확장 설명:**
1. 시장 기회: 글로벌 고객 지원 시장 연 12% 성장, AI 도입률 30% 미만
2. 고객 니즈: 응답 속도는 중요하지만 오버헤드 없이 도입하고 싶음
3. 우리의 능력: 사전 학습된 모델 + 한국 고객 특화 (한글 NLP, 전자계약법 준수)

**Product Trio 검증:**
- **PM**: TAM 충분함 (글로벌 CS 팀 100만 명). 경쟁사는 먼저 시장 점유했으나, UX 만족도 낮음 (기회)
- **Designer**: "코딩 없이"가 차별점이므로 설정 UI가 매우 중요. 초기 15분 이내에 첫 챗봇을 배포할 수 있어야 함
- **Engineer**: 사전 학습 모델 + 파인튜닝으로 빠르게 구현 가능. 한글 NLP 라이브러리(Korean BERT, KoGPT) 활용으로 경쟁사 대비 20% 빠른 배포
